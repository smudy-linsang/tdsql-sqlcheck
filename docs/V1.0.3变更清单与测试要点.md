## 一、新增功能（5项）

### 1.1 数据库上线检查模块（替代tdsql_12.sh脚本）

**变更描述**：将原有的 `tdsql_12.sh` 命令行巡检脚本的12项检查功能完整迁移至Web系统，在"实例与体检"菜单下新增"上线检查"子页面。

**新增文件**：
- `backend/engine/schema_inspector.py`（295行）— 12项Schema检查引擎

**修改文件**：
- `backend/api/inspection.py` — 新增 `POST /api/v1/inspection/schema-check` 端点
- `backend/models/__init__.py` — 新增 `SchemaCheckRequest` 请求模型
- `backend/services/auth_service.py` — `ALL_MENU_KEYS` 和 `MENU_LABELS` 添加 `schema-check`
- `backend/services/database.py` — role_permissions迁移脚本，为已有角色自动添加 `schema-check` 菜单权限
- `frontend/index.html` — 新增侧边栏菜单项 + 上线检查页面（实例选择器+12项折叠面板+KPI统计卡片）
- `frontend/static/js/app.js` — 新增 `schemaCheckConnId`/`schemaCheckResults`/`schemaCheckSummary`/`schemaCheckLoading` 状态变量 + `runSchemaCheck()` 函数

**12项检查清单**：

| 检查ID | 检查项 | 严重级别 | 检查内容 |
|--------|--------|---------|---------|
| C01 | 字符编码非utf8mb4的表 | ERROR | 表级collation不以utf8mb4开头 |
| C02 | 字符编码非utf8mb4的列 | ERROR | 列级collation不以utf8mb4开头 |
| C03 | 大小写敏感未设置的表 | WARNING | 表collation不是utf8mb4_bin |
| C04 | 大小写敏感未设置的列 | WARNING | 列collation不是utf8mb4_bin |
| C05 | 表名超过32个字符 | WARNING | LENGTH(table_name) >= 32 |
| C06 | 索引数量>=5 | WARNING | 单表索引过多影响写入性能 |
| C07 | 无主键的表 | ERROR | BASE TABLE无PRIMARY KEY |
| C08 | varchar字段长度>500 | WARNING | CHARACTER_MAXIMUM_LENGTH > 500 |
| C09 | 无注释的列 | INFO | column_comment为空 |
| C10 | 无注释的表 | INFO | table_comment为空 |
| C11 | 字段数>50的表 | WARNING | 单表字段过多 |
| C12 | timestamp类型字段 | WARNING | TDSQL建议用datetime替代timestamp |

**测试要点**：
- SIT：连接实例后执行检查，验证12项均有结果返回；各检查项SQL正确性
- UAT：菜单可见性（admin/dba可见，developer/auditor不可见）；结果折叠面板交互；KPI统计准确性
- 数据源：通过 `information_schema` 查询目标实例的表/列/索引元数据

### 1.2 HTML报告导出功能

**变更描述**：上线检查结果支持导出为独立HTML格式报告文件，可离线查看和分发。

**修改文件**：
- `backend/api/inspection.py` — 新增 `POST /api/v1/inspection/schema-check/report` 端点，返回 `HTMLResponse`
- `frontend/index.html` — KPI统计卡片行末尾新增"导出报告"按钮（Download图标）
- `frontend/static/js/app.js` — 新增 `exportSchemaCheckReport()` 函数，通过Blob下载

**报告内容**：
- 渐变色标题头（实例名称+检查时间+12项检查标识）
- 5项统计摘要卡片（问题总数/ERROR/WARNING/INFO/通过项）
- 12项检查结果详情（严重级别标签+检查名+问题数+修复建议+问题明细表格）
- 页脚（工具版本+生成时间）

**测试要点**：
- SIT：调用report接口返回200+HTML内容；HTML可正常渲染；12项结果完整
- UAT：点击"导出报告"按钮后浏览器自动下载；文件名格式 `上线检查报告_YYYY-MM-DD.html`；离线打开HTML内容正确

### 1.3 实例管理编辑功能

**变更描述**：实例管理模块新增"编辑"功能，支持修改已添加实例的全部连接信息。

**新增接口**：

### 1.4 项目管理删除与停用功能分离

**变更描述**：原"删除"按钮仅将项目状态改为inactive（软删除），现拆分为两个独立功能：真正删除（物理删除）和停用/启用切换。

- `frontend/index.html` — 操作列新增"停用/启用"按钮；列宽80→160；按钮文字根据状态动态显示
- `frontend/static/js/app.js` — 新增 `toggleProjectStatus()` 函数；`deleteProject()` 确认提示改为"删除后不可恢复"

**测试要点**：
- SIT：DELETE接口真正删除记录（数据库验证）；PUT toggle接口正确切换状态；404验证
- UAT："停用"按钮将active改为inactive，列表状态标签变化；"启用"按钮恢复；"删除"按钮弹出确认框，确认后记录消失

## 三、缺陷修复（4项）

### 3.1 耗时阈值单位标签修正

**变更描述**：前端"耗时阈值"标签显示为"ms"，但后端实际使用秒为单位（`AVG_TIMER_WAIT/1e12 >= min_time`），标签与实际不符。

**修改文件**：
- `frontend/index.html` — 3处 `耗时阈值(ms)` → `耗时阈值(秒)`，1处 `阈值(ms)` → `阈值(秒)`

**影响范围**：扫描任务抽屉表单、扫描计划表单、扫描计划列表

**测试要点**：
- UAT：所有"耗时阈值"相关标签显示为"秒"而非"ms"；输入0.1表示≥0.1秒（100ms）

### 3.2 实例选择器选中后显示连接名

**变更描述**：实例选择器（el-select）选中后显示连接ID（如`5ea70d74`）而非连接名称，原因是 `el-option` 缺少 `:label` 属性。

**修改文件**：
- `frontend/index.html` — 3处实例选择器的 `el-option` 补充 `:label` 属性：
  - 扫描任务抽屉"目标实例"：`label="c.name + ' (' + c.host + ':' + c.port + '/db)'"`
  - 数据库体检"选择实例"：`label="c.name + ' (' + c.host + ':' + c.port + ')'"`
  - 大表治理"选择实例"：`label="c.name + ' (' + c.host + ':' + c.port + ')'"`

**测试要点**：
- UAT：选中实例后下拉框显示连接名称（如"SIT-分布式实例A (119.45.220.89:15005)"）而非ID串码

### 3.3 上线检查菜单权限修复

**变更描述**：新增的"上线检查"菜单在所有角色下不可见，原因是 `ALL_MENU_KEYS`、`MENU_LABELS` 和 `role_permissions` 表中缺少 `schema-check`。

**修改文件**：
- `backend/services/auth_service.py` — `ALL_MENU_KEYS` 列表和 `MENU_LABELS` 字典添加 `schema-check`
- `backend/services/database.py` — 初始化脚本新增迁移SQL，为所有已有角色自动插入 `schema-check` 权限记录（visible=1）

**测试要点**：
- SIT：`GET /api/v1/auth/visible-menus` 返回包含 `schema-check`
- UAT：admin/dba角色登录后侧边栏"实例与体检"下可见"上线检查"子菜单

### 3.4 实例连接请求字段名修复

**变更描述**：`TDSQLConnectRequest` Pydantic模型中字段名为 `user`，但前端发送的是 `username`，导致422验证错误。同时缺少 `is_distributed` 字段。

**修改文件**：
- `backend/api/tdsql_manage.py` — `TDSQLConnectRequest.user` → `username`；新增 `is_distributed: bool = True` 字段；相关端点代码同步更新

**测试要点**：
- SIT：POST `/api/v1/tdsql/connections` 发送 `username` 字段返回200；发送 `user` 字段返回422

## 十、已知限制与注意事项

1. **上线检查依赖实例连接**：执行检查前需先在"实例管理"中连接目标实例
2. **HTML报告为静态生成**：每次导出重新执行12项检查并生成报告，非缓存上次结果
3. **项目物理删除不可恢复**：删除项目后关联的规则集绑定关系将失效（规则集本身不删除）
4. **performance_schema依赖**：上线检查通过 `information_schema` 查询元数据，需确保目标实例可正常访问系统表
5. **实例编辑密码字段**：编辑时密码输入框为空，需重新输入密码才能保存（不支持保留原密码）
