## 一、接口总览

### 1.1 接口分类

| 类别 | 接口数量 | 说明 |
|-----|---------|------|
| SQL审核API | 6 | 单条/文件/上传审核，审核历史，结果对比 |
| 慢SQL分析API | 10 | 慢SQL管理/分析/指纹/死锁/长事务/字符集诊断/索引推荐/改写建议/效果验证 |
| 大表治理API | 8 | 大表盘点/分区监控/保留期/分区建议/清理建议/变更管控/巡检 |
| 质量门禁API | 5 | 门禁规则CRUD + 门禁评估 |
| 规则管理API | 6 | 规则列表/详情/启用禁用/白名单/分类视图 |
| GitLab集成API | 3 | MR Webhook + Diff审核 + 仓库审核 |
| 监控告警API | 4 | 指标查询/告警列表/告警规则CRUD |
| 巡检报告API | 4 | 巡检任务/报告列表/详情/下载 |
| 项目管理API | 5 | 项目CRUD + 项目统计 |
| TDSQL管理API | 14 | 连接管理/元数据/慢SQL抓取/Proxy配置/字符集检查/大表检查/SET发现 |
| Dashboard API | 4 | 统计概览/趋势/规则排名/质量评分 |
| 认证与健康 | 3 | 登录/APIKey验证/健康检查 |
| **合计** | **72** | |

### 1.2 通用约定

**Base URL:** `http://<host>:8000/api/v2`

**认证方式:** Header `X-API-Key: <key>`（管理类接口需要认证，审核类接口可选认证）

**统一响应格式:**

```json
{
  "code": 0,
  "message": "success",
  "data": { ... },
  "timestamp": "2026-06-17T12:00:00Z"
}
```

**分页参数:**

| 参数 | 类型 | 默认值 | 说明 |
|-----|------|-------|------|
| page | int | 1 | 页码 |
| page_size | int | 20 | 每页数量（最大100） |

**分页响应:**

```json
{
  "code": 0,
  "data": {
    "items": [...],
    "total": 150,
    "page": 1,
    "page_size": 20
  }
}
```

**错误码体系:**

| 错误码 | HTTP | 含义 |
|-------|------|------|
| E0001 | 422 | SQL解析失败 |
| E0002 | 500 | 规则执行异常 |
| E0003 | 503 | TDSQL连接失败 |
| E0004 | 504 | TDSQL查询超时 |
| E0006 | 401 | API认证失败 |
| E0007 | 403 | 权限不足 |
| E0008 | 404 | 资源不存在 |
| E0009 | 422 | 参数校验失败 |
| E0010 | 500 | SQLite写入失败 |
| E0011 | 500 | 报告生成失败 |
| E0012 | 500 | 门禁评估失败 |

## 二、SQL审核API

### 2.1 单条SQL审核

```
POST /audit/sql
```

**请求体:**

```json
{
  "sql": "SELECT * FROM t_order WHERE user_id = 1001",
  "project_id": "proj_001",
  "connection_id": "conn_001",
  "enable_metadata": true
}
```

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| sql | string | 是 | SQL语句 |
| project_id | string | 否 | 项目ID（关联规则集和门禁策略） |
| connection_id | string | 否 | TDSQL连接ID（用于元数据增强审核） |
| enable_metadata | boolean | 否 | 是否启用元数据增强（需connection_id） |

**响应:**

```json
{
  "code": 0,
  "data": {
    "sql": "SELECT * FROM t_order WHERE user_id = 1001",
    "sql_type": "SELECT",
    "passed": false,
    "violations": [
      {
        "rule_id": "R012",
        "category": "dml",
        "severity": "ERROR",
        "message": "禁止使用SELECT *",
        "suggestion": "请指定具体字段名，如SELECT col1, col2 FROM ...",
        "line_number": 1
      }
    ],
    "gate_result": {
      "passed": false,
      "error_count": 1,
      "warning_count": 0,
      "blocked_by": ["R012"]
    }
  }
}
```

## 三、慢SQL分析API

### 3.1 慢SQL列表

```
GET /slow-queries?page=1&page_size=20&connection_id=conn_001&sort=total_time&order=desc
```

| 参数 | 类型 | 说明 |
|-----|------|------|
| connection_id | string | TDSQL连接ID |
| sort | string | 排序字段: total_time/exec_count/max_time/rows_examined |
| order | string | asc/desc |
| status | string | pending/analyzed/optimized/closed |

## 十五、CLI接口

### 15.1 命令格式

```bash
tdsql-sqlcheck <command> [options]
```

### 15.2 命令列表

| 命令 | 说明 | 示例 |
|-----|------|------|
| audit | 审核SQL文件 | `tdsql-sqlcheck audit -f migration.sql` |
| audit-stdin | 从stdin审核 | `cat query.sql \| tdsql-sqlcheck audit-stdin` |
| version | 版本信息 | `tdsql-sqlcheck version` |
| list-rules | 列出所有规则 | `tdsql-sqlcheck list-rules --category ddl` |

### 15.2 audit命令选项

| 选项 | 类型 | 默认值 | 说明 |
|-----|------|-------|------|
| --file, -f | string | 必填 | SQL文件路径 |
| --format | json\|sarif\|html\|pdf | json | 输出格式 |
| --gate | strict\|normal\|loose | 无 | 门禁策略 |
| --config, -c | string | 无 | 配置文件路径 |
| --output, -o | string | stdout | 输出文件路径 |
| --metadata | boolean | false | 是否启用元数据增强（需--connection） |
| --connection | string | 无 | TDSQL连接配置ID |

### 15.3 退出码

| 退出码 | 含义 |
|-------|------|
| 0 | 审核通过（门禁通过或未启用门禁） |
| 1 | 审核未通过（门禁阻断） |
| 2 | 系统错误（文件不存在、解析失败等） |
