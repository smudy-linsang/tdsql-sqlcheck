> **更新类型**: P0 级严重交互缺陷修复 (RBAC 403 强框消除与无感权限路由)  
> **版本号**: `v1.2.0.6`  
> **发布日期**: 2026-07-24  

---

## 一、核心修复与优化内容说明

### 1. 彻底解决多角色登录/首次登录强制改口令时的“无权执行”弹窗霸屏问题 (P0)
- **原因分析**：在之前的版本中，前端底层网络拦截器 `apiFetch` 包含统一硬拦截，凡后端返回 HTTP 403 (Forbidden) 就会弹出一条 Warning 消息提示。当具有部分权限的角色（如开发人员或审计员）登录时，后台并发发起的 5~10 个只读数据请求被 403 拒后，导致屏幕中央瞬间连弹 10 条黄色消息框，造成极差的用户体验。
- **修复方案**：
  - 移除了 `apiFetch` 中的 403 强行 Toast 弹窗逻辑。
  - 数据加载逻辑重构为基于 `visibleMenus`（角色可访问菜单集合）的动态数据加载，仅针对当前账户拥有的模块进行请求。
  - 增加了首屏智能路由重定向：用户登录后，若默认页面不在其可访问列表中，系统将自动跳转至其拥有的第一个有效工作台页面。
  - 强制修改密码期间挂起后台数据请求，直到密码修改完成。

## 二、部署与升级步骤

### 1. 解压安装包
```bash
sudo tar -xzf tdsql-sqlcheck-v1.2.0.6-linux-x86_64.tar.gz -C /opt/
cd /opt/tdsql-sqlcheck-v1.2.0.6-linux-x86_64
```

### 2. 执行安装升级脚本
```bash
cp deploy/env.template deploy/.env
sudo ./deploy/install.sh
```

### 3. 服务健康状态验证
```bash
curl http://localhost:8000/health
```
**期望返回**：
`{"status":"ok","version":"1.2.0.6"}`
