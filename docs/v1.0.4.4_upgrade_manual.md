## 二、本次更新内容 (v1.0.4.3 → v1.0.4.4)

本次更新优化了 `monitordb` 扫描数据源在大集群多租户/多数据库环境下的过滤机制：

### 1. 扫描结果按实例连接指定的数据库进行过滤
* **问题现象**：在使用“全网慢SQL(monitordb)”数据源创建扫描任务时，系统虽然在实例管理中连接了指定数据库的实例（如 `olp`），但扫描抓取时会无差别地将该 TDSQL 集群上**所有数据库**的慢 SQL 全部拉取并分析出来。在大规模、多数据库并存的集群里，非当前数据库的诊断报告堆积在一起，极大干扰了开发和运维查找指定库问题的效率。
* **优化内容**：
  - 更新扫描范围过滤：在 [scan_service.py](file:///c:/TDSQL_SQLCHECK/TDSQL-SQLCheck/backend/services/scan_service.py) 中，当用户通过某个实例连接且数据源为 `monitordb` 触发慢 SQL 扫描时，后端会自动提取该连接配置的默认数据库名 `db_name`。
  - 精确库过滤：将该 `db_name` 作为过滤条件传递至底层 `get_cluster_slow_queries(..., database=db_name)`。底层 SQL 拼接时将精确附带 `AND db = <db_name>`（或 `SCHEMA_NAME = <db_name>`），从而使扫描任务**仅抓取、列出和分析当前实例连接中指定的这一个数据库**的慢 SQL。
  - 未指定数据库降级：若连接中未配置默认数据库，则保持不进行库名过滤的兼容行为（拉取全集群慢 SQL）。

## 四、升级步骤（严格按顺序执行）

### 第 1 步：上传并校验发布包
将发布包 `tdsql-sqlcheck-v1.0.4.4-linux-x86_64.tar.gz` 拷贝至目标服务器的 `/tmp/dist/` 目录下：
```bash
cd /tmp/dist/
sha256sum -c tdsql-sqlcheck-v1.0.4.4-linux-x86_64.tar.gz.sha256
```
**期望输出**：`tdsql-sqlcheck-v1.0.4.4-linux-x86_64.tar.gz: OK`
*若校验失败，请立即停止部署并重新传输文件。*

### 第 2 步：解压发布包
```bash
tar -xzf tdsql-sqlcheck-v1.0.4.4-linux-x86_64.tar.gz
cd tdsql-sqlcheck-v1.0.4.4-linux-x86_64
ls -la
```
确认存在 `backend/`、`frontend/`、`deploy/`、`wheels/`、`requirements.txt`、`VERSION` 等文件和目录。

### 第 3 步：停止运行中的旧服务
```bash
sudo systemctl stop tdsql-sqlcheck
sudo systemctl status tdsql-sqlcheck
```
确保服务已停止且状态为 `inactive (dead)`。

### 第 4 步：备份旧版本
```bash
sudo cp -a /opt/tdsql-sqlcheck /opt/tdsql-sqlcheck-v1.0.4.3-backup
ls -la /opt/tdsql-sqlcheck-v1.0.4.3-backup/
```
确保备份完整，便于在升级失败时秒级回滚。

### 第 5 步：覆盖更新文件 (保留配置与临时数据)
```bash
# 复制后端核心代码
sudo cp -r backend/ /opt/tdsql-sqlcheck/

# 复制前端核心静态包
sudo cp -r frontend/ /opt/tdsql-sqlcheck/

# 复制依赖说明文件
sudo cp requirements.txt /opt/tdsql-sqlcheck/

# 复制版本标记文件
sudo cp VERSION /opt/tdsql-sqlcheck/

# 复制新版运维与验证脚本
sudo cp -r deploy/ /opt/tdsql-sqlcheck/
```
> [!IMPORTANT]
> **请勿覆盖** `/opt/tdsql-sqlcheck/.env` 文件以及 `/opt/tdsql-sqlcheck/data/` 目录，以保持原有的数据库连接设置与口令！

### 第 6 步：升级 Python 离线依赖包 (利用内置 wheels)
```bash
cd /opt/tdsql-sqlcheck
sudo ./venv/bin/pip install --no-index --find-links=/tmp/dist/tdsql-sqlcheck-v1.0.4.4-linux-x86_64/wheels/ -r requirements.txt
```
**期望输出**：`Successfully installed` 或 `already satisfied`。

### 第 7 步：启动服务
```bash
sudo systemctl start tdsql-sqlcheck
sleep 5
sudo systemctl status tdsql-sqlcheck
```

### 第 8 步：执行部署验证脚本
```bash
cd /opt/tdsql-sqlcheck
sudo bash deploy/verify_deploy.sh
```
**期望输出**：
```text
════ 部署验证 v1.0.4.4 @ http://127.0.0.1:8000 ════
  [PASS] 健康检查 /health
  [PASS] 版本号 1.0.4.4
  [PASS] 首页可访问
  ...
════ 验证结果: PASS=7 FAIL=0 ════
部署验证全部通过 ✔
```
