## 1. 版本定位

v1.0.2 是本系统**首个生产上线版本**。此前的内部迭代版号（V1.0/V2.0/V2.1、前端 V3.0）为研发过程版本，自本次上线起，对外版本号统一从 **v1.0.2** 起算，后续按 v1.0.x（补丁）/ v1.x（特性）演进。程序内 `/health`、`/api/v1/admin/info`、页面均显示 1.0.2。

## 2. 功能全景（本版本交付能力）

### 2.1 SQL 审核核心
- **119 条审核规则 / 9 大分类**：命名5、DDL22、DML9、性能5、分布式14、索引10、事务4、安全8、**Oracle迁移兼容42**；
- Oracle迁移兼容规则依据 TDSQL 原厂《TDSQL兼容业务系统适配改造方案 V1.5.1》全量落地（to_char/nvl/decode/rownum/merge into/with as/窗口函数/分片键军规等 52 个规范点，每条带原厂改写建议）；
- 审核入口 5 个：即时审核、SQL/MyBatis-XML 文件审核、文件上传、GitLab Diff/仓库/Webhook、元数据增强审核（取真实分片键/索引提升精度）；
- 审核报告：HTML/PDF 导出，历史留存可追溯。

### 2.2 平台治理
- **多项目多租户**：项目绑定规则集（按项目启停/调级规则）与质量门禁（ERROR/WARNING 阈值、预设 strict/loose 策略），门禁结果随审核返回；
- **慢SQL治理闭环**：多实例扫描（digest/processlist）、六维分析、状态流转（待处理/已优化/已忽略）、EXPLAIN 执行计划分析、定时扫描计划（leader 租约防重）；
- **实例与体检**：数百 TDSQL 实例纳管（连接注册表+池化+口令加密存储）、字符集一致性体检、大表治理分级（L1/L2/L3）、巡检任务管理；
- **监控告警**：业务指标阈值告警与确认闭环，顶栏实时提醒。

### 2.3 安全与合规
- 登录认证（PBKDF2 口令 + HMAC 自包含令牌）、**四角色 RBAC**（admin/dba/developer/auditor）+ 角色权限矩阵管理；
- 连续登录失败锁定（5 次/15 分钟）、首登强制改密、口令复杂度校验；
- SQL 文本入库脱敏（字面量→?）、操作审计日志（含 IP）、数据保留策略自动清理；
- 纯内网运行：前端资产全部本地化，零外网请求。

### 2.4 可观测与运维
- `/health` 健康检查、`/metrics` Prometheus 指标；
- 一键部署/验证/回滚脚本套件（麒麟 V10 SP3 适配、全离线安装）。

## 4. 质量声明

- 全量自动化回归：**880 通过 / 0 失败 / 55 跳过**（含 Oracle 兼容规则 103 个专项用例）；
- 历经 SIT/UAT 各多轮 + 独立质检验收（详见归档的验收报告）；
- 性能基线：单条 SQL 审核 0.60ms（119 规则全开）；扫描并发限流保护目标库。

## 5. 已知限制

1. 元数据增强审核、EXPLAIN(SQL直连) 依赖目标实例网络可达与账号权限（见部署手册 3.3）；
2. TDSQL 分布式 Proxy 的 performance_schema 不支持时间窗过滤，扫描时间窗口作为任务元数据记录；
3. 审核为静态分析＋可选元数据增强，不执行业务 SQL（安全设计使然）；`insert into … select` 目标表自增列/分区限制等少数规则为提示级，需人工复核；
4. 单机部署（本版本容量已覆盖数百实例场景）；多活部署调度器已具备 leader 租约，但未做前端会话共享验证，暂不建议。

## 6. 升级与回滚

首次上线无升级路径。回滚即卸载（部署手册第 9 节）。后续版本经同一发布包流程升级，支持分钟级 `rollback.sh` 回退，元数据库结构增量兼容。
