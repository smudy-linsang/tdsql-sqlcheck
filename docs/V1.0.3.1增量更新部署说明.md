## 四、部署步骤（严格按顺序执行）

### 第1步：SSH登录目标服务器

```bash
ssh root@10.243.16.238
```

### 第2步：进入发布包目录并校验

```bash
cd /tmp/dist
ls -lh tdsql-sqlcheck-v1.0.3.1-linux-x86_64.tar.gz*
sha256sum -c tdsql-sqlcheck-v1.0.3.1-linux-x86_64.tar.gz.sha256
```

**期望输出**: `tdsql-sqlcheck-v1.0.3.1-linux-x86_64.tar.gz: OK`

如果不是OK，**停止操作**，报告校验失败。

### 第3步：解压发布包

```bash
tar -xzf tdsql-sqlcheck-v1.0.3.1-linux-x86_64.tar.gz
cd tdsql-sqlcheck-v1.0.3.1-linux-x86_64
ls -la
```

确认目录内有 `backend/`、`frontend/`、`deploy/`、`wheels/`、`requirements.txt`、`VERSION` 等文件。

### 第4步：停止当前服务

```bash
sudo systemctl stop tdsql-sqlcheck
sudo systemctl status tdsql-sqlcheck
```

确认服务已停止（Active: inactive）。

### 第5步：备份当前版本

```bash
sudo cp -a /opt/tdsql-sqlcheck /opt/tdsql-sqlcheck-v1.0.3-backup
ls -la /opt/tdsql-sqlcheck-v1.0.3-backup/
```

确认备份目录存在且内容完整。

### 第6步：覆盖更新代码（保留 .env 和 data/）

```bash
# 覆盖后端代码
sudo cp -r /tmp/dist/tdsql-sqlcheck-v1.0.3.1-linux-x86_64/backend/ /opt/tdsql-sqlcheck/

# 覆盖前端代码
sudo cp -r /tmp/dist/tdsql-sqlcheck-v1.0.3.1-linux-x86_64/frontend/ /opt/tdsql-sqlcheck/

# 覆盖requirements.txt
sudo cp /tmp/dist/tdsql-sqlcheck-v1.0.3.1-linux-x86_64/requirements.txt /opt/tdsql-sqlcheck/

# 覆盖VERSION文件
sudo cp /tmp/dist/tdsql-sqlcheck-v1.0.3.1-linux-x86_64/VERSION /opt/tdsql-sqlcheck/

# 覆盖部署脚本（可选，供后续运维使用）
sudo cp -r /tmp/dist/tdsql-sqlcheck-v1.0.3.1-linux-x86_64/deploy/ /opt/tdsql-sqlcheck/
```

**重要**：不要覆盖 `.env` 文件和 `data/` 目录，保留原有配置和数据。

### 第7步：安装/更新Python依赖（离线）

```bash
cd /opt/tdsql-sqlcheck
sudo /opt/tdsql-sqlcheck/venv/bin/pip install --no-index --find-links=/tmp/dist/tdsql-sqlcheck-v1.0.3.1-linux-x86_64/wheels/ -r requirements.txt
```

**期望输出**: 所有依赖 `Successfully installed` 或 `already satisfied`。

### 第8步：启动服务

```bash
sudo systemctl start tdsql-sqlcheck
sleep 5
sudo systemctl status tdsql-sqlcheck
```

确认服务 Active: active (running)。

### 第9步：执行部署验证脚本

```bash
cd /opt/tdsql-sqlcheck
sudo bash deploy/verify_deploy.sh
```

**期望输出**: 全部 PASS，0 FAIL。

或手动验证：

```bash
# 健康检查（版本号应为1.0.3.1）
curl -s http://127.0.0.1:8000/health | python3 -m json.tool

# 前端可访问
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:8000/

# 静态资源
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:8000/static/js/app.js
```

## 七、重要注意事项

1. **不要修改 `.env` 文件**：v1.0.3 的配置文件完全兼容 v1.0.3.1
2. **不要操作数据库**：v1.0.3.1 无破坏性表结构变更，元数据库无需迁移
3. **保留备份**：`/opt/tdsql-sqlcheck-v1.0.3-backup` 不要删除，用于回滚
4. **大表治理测试**：如果测试数据库没有大表（>1GB），会显示"未发现大表"，这是正常现象。生产环境有大表时会正常显示。
5. **SSH执行**：所有命令均通过SSH在目标Linux服务器上执行，不要在Windows本地执行。
