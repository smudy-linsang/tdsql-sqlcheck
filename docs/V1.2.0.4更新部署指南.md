## 📋 二、开发侧全量变更说明 (部署智能体必读)

为了让内网部署智能体与运维人员全面了解本次发布的更改，以下为 V1.2.0.4 开发侧完成的核心架构重构与修补明细：

### 1. 🚀 高并发吞吐架构重构 (P0-1 & P0-3)
- **`async def` ➔ `def` 路由转换**：将 `backend/api/*.py` 下 122 个涉及数据库读写的 API 路由声明从 `async def` 改为 `def`。FastAPI 会自动将阻塞型数据库 IO 调度至 `anyio` 线程池并发执行，重叠远程数据库的网络往返等待（RTT），彻底解决单线程 Event Loop 被卡死、用户一多就假死的问题。
- **并发线程池容量调大**：系统启动 lifespan 时，自动将 `anyio` 线程池总容量提升至 `100` (`total_tokens = 100`)。

### 2. 🛢️ 元数据库连接池扩容与按需探活 (PERF-04)
- **连接池容量提升**：`SQLCHECK_DB_POOL_SIZE` 默认值由 10 调大至 **60**，完美匹配 Worker 线程池并发需求。
- **按需探活策略**：优化连接 checkout 逻辑，仅在连接底层 Socket 断开或在池中空闲超过 10 秒以上时才触发 `ping()`，高频热连接复用零额外 RTT 开销。

### 3. 🧹 彻底清除硬编码假 IP & 动态解析真实 Host/节点
- **假 IP 剥离**：彻底删除了原代码中写死的 `10.0.8.21`、`10.0.8.22`、`10.0.8.23` 虚假服务器 IP。
- **真实节点解析**：服务器级别巡检重构为动态从绑定的 TDSQL 实例注册信息 `pool.config.host` 及集群节点列表中解析真实 IP/主机名，并优先抽取原厂 `monitordb`（`tdsqlpcloud_monitor`）监控底座。

### 4. 🛡️ 审计日志 Task 内存防 GC 保护 (PERF-02)
- 中间件 `AuthMiddleware` 增加了全局 `_BACKGROUND_TASKS` 强引用集合与完成回调，确保异步写审计日志 100% 可靠落库，避免高并发下 Task 被 Python 垃圾回收器提前 GC。

### 5. ⚡ 跨 Worker 权限版本号驱动 0ms 实时失效 (PERF-03)
- 在系统数据库 `system_config` 表中引入 `permission_version`。角色权限变更时自动递增版本号，各 Worker 实时校验，实现多进程/多 Worker 部署下的权限变更 0ms 实时生效。

### 6. 🐛 巡检冗余循环修补 (BUG-PERF-01)
- 修正了 `daily_inspect_service.py` 中 `run_server_daily` 与 `conn.commit()` 被误写在节点循环体内的 Bug，恢复为每次巡检单事务统一提交 1 次。

## 🧹 三、历史假 IP 脏数据清洗（升级关键步骤）

由于先前版本（v1.2.0.3 及更早）曾在数据库 `server_daily_inspection` 表中写入了 `10.0.8.21 ~ 23` 的测试脏数据，**在升级到 v1.2.0.4 后必须执行以下数据清洗步骤**。

### 1. 手动 SQL 清洗指令（MySQL / MariaDB 元数据库）
```sql
DELETE FROM server_daily_inspection WHERE ip IN ('10.0.8.21', '10.0.8.22', '10.0.8.23');
COMMIT;
```

### 2. 自动化一键 Python 清洗命令行（脚本嵌入）
```bash
cd /opt/tdsql-sqlcheck && source venv/bin/activate
python -c "import sys, os; sys.path.insert(0, '.'); from backend.services.database import _get_connection; conn = _get_connection(); cur = conn.execute(\"DELETE FROM server_daily_inspection WHERE ip IN ('10.0.8.21', '10.0.8.22', '10.0.8.23')\"); conn.commit(); print('已成功清洗历史假 IP 脏数据!')"
```

## 🛠️ 四、极速一键部署升级命令与自动化脚本

### 1. 标准产品快速解压部署命令
```bash
cd dist && sha256sum -c tdsql-sqlcheck-v1.2.0.4-linux-x86_64.tar.gz.sha256
tar -xzf tdsql-sqlcheck-v1.2.0.4-linux-x86_64.tar.gz -C /opt/
cd /opt/tdsql-sqlcheck-v1.2.0.4-linux-x86_64
cp deploy/env.template deploy/.env
sudo ./deploy/install.sh
```

### 2. 自动化 Shell 脚本 (含脏数据清洗)

```bash
#!/usr/bin/env bash
# ==============================================================================
# TDSQL-SQLCheck V1.2.0.4 一键部署与升级自动化脚本
# ==============================================================================
set -e

APP_DIR="/opt/tdsql-sqlcheck"
RELEASE_TAR="/tmp/tdsql-sqlcheck-v1.2.0.4-linux-x86_64.tar.gz"

echo "==== [Step 1/6] 备份旧版本并停止服务 ===="
if [ -d "${APP_DIR}" ]; then
    sudo cp -a ${APP_DIR} ${APP_DIR}-backup-$(date +%Y%m%d%H%M) || true
fi
sudo systemctl stop tdsql-sqlcheck || pkill -f "uvicorn backend.main:app" || true

echo "==== [Step 2/6] 部署代码解压 ===="
sudo mkdir -p ${APP_DIR}
if [ -f "${RELEASE_TAR}" ]; then
    sudo tar -xzf ${RELEASE_TAR} -C ${APP_DIR} --strip-components=1
else
    echo "[Info] 未发现 Release 压缩包，按当前代码目录覆盖部署"
fi

cd ${APP_DIR}

echo "==== [Step 3/6] 离线 Python 虚拟环境准备 ===="
if [ ! -d "venv" ]; then
    python3 -m venv venv
fi
source venv/bin/activate

if [ -d "wheels" ]; then
    pip install --no-index --find-links=wheels -r requirements.txt || true
elif [ -d "../wheels" ]; then
    pip install --no-index --find-links=../wheels -r requirements.txt || true
else
    pip install -r requirements.txt || true
fi

echo "==== [Step 4/6] 执行假 IP 脏数据清洗与数据库 Schema 初始化 ===="
python -c "import sys, os; sys.path.insert(0, '.'); from backend.services.database import ensure_db, _get_connection; ensure_db(); conn = _get_connection(); cur = conn.execute(\"DELETE FROM server_daily_inspection WHERE ip IN ('10.0.8.21', '10.0.8.22', '10.0.8.23')\"); conn.commit(); print('[Data Clean] 假 IP 脏数据清理成功!')"

echo "==== [Step 5/6] 启动 Backend 高并发服务 (4 Worker) ===="
nohup python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 --workers 4 > backend_restart.log 2>&1 &

echo "==== [Step 6/6] 健康检查验货 ===="
sleep 3
HEALTH_CHECK=$(curl -s http://127.0.0.1:8000/health || echo "FAIL")
echo "健康探针返回: ${HEALTH_CHECK}"

if [[ "${HEALTH_CHECK}" == *"1.2.0.4"* ]]; then
    echo "🎉 [Success] TDSQL SQL审核工具成功升级/部署至 V1.2.0.4 稳定定版！"
else
    echo "❌ [Error] 启动失败，请检查 backend_restart.log"
    cat backend_restart.log | tail -n 20
    exit 1
fi
```
