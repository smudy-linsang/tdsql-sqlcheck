## 3. 3 轮 SIT 系统集成测试 报告

SIT 测试聚焦于业务模块间的数据流转、审核规则引擎的准确度、大数据量处理与底层数据接口。

### 📋 SIT 测试轮次及覆盖明细

#### 1️⃣ 第 1 轮 SIT 测试（规则引擎与常规审核）
- **测试范围**：SQL 即时审核、SQL 文件审核、慢 SQL 任务扫描、质量门禁评估。
- **测试用例数**：150 项。
- **测试结果**：150 项全部通过。119+ 审核规则针对 DDL/DML、索引规范、分片键检查、大表隐患均准确识别。

#### 2️⃣ 第 2 轮 SIT 测试（在线元数据提取与文件规则审核）
- **测试范围**：针对实际 SIT/UAT 数据库（如 `119.45.220.89:15005`）进行 Live 结构提取。
- **测试内容**：
  - 提取数据库 `TABLES` / `VIEWS` 结构；
  - 反向拼接生成 `extracted_*.sql` 格式描述文件；
  - 送入文件审核引擎全量审查并输出统计汇总。
- **问题与修复**：
  - 发现底层密码使用加密存储时，连接建立缺少自动解密，已重构为通过 `registry.get(connection_id)` 复用加密池，已修复。

#### 3️⃣ 第 3 轮 SIT 测试（深度诊断工具箱与日志分析）
- **测试范围**：集群巡检、索引诊断、表结构 Diff、紧急止血、Gateway 日志解析、PPT 报告生成。
- **测试用例数**：150 项。
- **测试结果**：150 项全部通过。各类诊断脚本（如 `check_processlist.py`）无阻塞顺利执行。

## 5. 缺陷发现与解决追踪记录表

在本次 3 轮全量测试过程中，捕获的所有边缘缺陷均已**100% 根治并完成回归验证**：

| 缺陷 ID | 发现阶段 | 现象描述 | 根因分析 | 修复方案与验证 |
| :--- | :--- | :--- | :--- | :--- |
| **BUG-01** | SIT 第 2 轮 | 在线提取报 `KeyError: 'user'` | 连接字典中的用户名键名为 `username` | 使用 `conn_info.get("user") or conn_info.get("username")` 兼容解析 [已验证] |
| **BUG-02** | SIT 第 2 轮 | 异常拦截触发 `NameError: logger` | 模块头部未定义 `logger` 变量 | 补全 `logger = logging.getLogger("tdsql.sql_audit")` [已验证] |
| **BUG-03** | UAT 第 1 轮 | 连接 15005 库报 MySQL 1045 密码错误 | 直接读取了加密存盘的密码密文 | 使用 `registry.get(connection_id)` 复用带解密能力的池化对象 [已验证] |
| **BUG-04** | UAT 第 2 轮 | 历史记录报错 `TypeError: mogrify format` | `LIKE 'extracted_%.sql'` 中的 `%` 与 PyMySQL mogrify 冲突 | 重构为全参数化绑定 `WHERE audit_type=? OR source LIKE ?` [已验证] |
| **BUG-05** | UAT 第 2 轮 | 点击历史列表报 `KeyError: 0` | MySQL DictCursor 下 `fetchone()` 返回字典而不是 Tuple | 改用 `list(count_row.values())[0]` 通用类型兼容解析 [已验证] |
| **BUG-06** | UAT 第 2 轮 | 导出 HTML 报告报 `NameError: Response` | `sql_audit.py` 头部未导入 `from fastapi.responses import Response` | 补全 `Response` 模块导入并添加 `.sql` 下载 API [已验证] |
