## 二、本次更新内容 (v1.1.0.0 → v1.1.0.1)

本次更新针对第一轮测试质检中暴露出的 8 项缺陷（BUG-01 ~ BUG-08）进行了全量整改，并通过了第二轮 100% 回归测试：

### 1. 越权漏洞堵塞 (BUG-01)
* **说明**：重构了 `check_permission` 鉴权算法。采用**前缀长度降序排序的最长前缀匹配**，并结合**路径边界检测**（`path == prefix or path.startswith(prefix + "/")`）。
* **效果**：彻底消除了由类似 `/api/v1/gate`（质量门禁）与 `/api/v1/gateway-log`（网关日志）等部分单词前缀冲突，或者 `/api/v1/slow-queries` 与 `/api/v1/slow-queries/analyze-explain` 嵌套子路径引起的遮蔽（shadowing）漏洞，确保非 admin 内置角色子权限接口级控制 100% 生效。

### 2. 首次登录强制改密闭环 (BUG-02)
* **说明**：
  * **前端**：锁定改密对话框，禁止点击空白处及按 ESC 键关闭，隐藏“取消”按钮。
  * **后端**：在全局中间件中进行强拦截。对于 `must_change_password=1` 的用户，除放行修改口令、退出登录、自我信息接口外，其余所有业务接口直接拦截并返回 `403`。

### 3. 多语句即时审核漏报修复 (BUG-03)
* **说明**：在即时 SQL 审核接口内部引入 `split_sql_statements` 语句分割逻辑，对多条 SQL 语句进行拆分并逐一进行深度规则审核。
* **效果**：汇总所有语句的违规信息，在违规消息前缀注明 `[第x条语句]` 以精确指引修改，消除了多语句合并输入时只审第一条、其余静默漏报的隐患。

### 4. 离线 HTML 对比报告渲染 (BUG-04)
* **说明**：将导出的 HTML 巡检比对报告的外部 CDN 依赖剔除，指向本地 `/static/vendor/echarts.min.js`，图表绘制重构为纯 ECharts 容器及配置。
* **效果**：在完全断网的内网隔离环境下也能完美渲染出性能指标折线图与对比大屏。

### 5. 实例修改即时生效 (BUG-05 & BUG-08)
* **说明**：在编辑保存连接配置后，自动执行物理断开连接并销毁连接池缓存。同时，断开连接操作会同步更新数据库表 `tdsql_connections.status` 字段为 `disconnected`。
* **效果**：确保下次访问自动利用新配置建立物理连接，状态展示与页面实时同步。

### 6. 可用性与测试用例订正 (BUG-06 & BUG-07)
* **说明**：在连接实例按钮上添加 loading 加载动画；修正测试用例的业务引导，使多库审核场景与实际承载该能力的“上线检查”模块对齐。

## 四、升级步骤（严格按顺序执行）

### 第 1 步：上传并校验发布包
将发布包 `tdsql-sqlcheck-v1.1.0.1-linux-x86_64.tar.gz` 拷贝至目标服务器的 `/tmp/dist/` 目录下：
```bash
cd /tmp/dist/
sha256sum -c tdsql-sqlcheck-v1.1.0.1-linux-x86_64.tar.gz.sha256
```
**期望输出**：`tdsql-sqlcheck-v1.1.0.1-linux-x86_64.tar.gz: OK`

### 第 2 步：解压发布包
```bash
tar -xzf tdsql-sqlcheck-v1.1.0.1-linux-x86_64.tar.gz
cd tdsql-sqlcheck-v1.1.0.1-linux-x86_64
```

### 第 3 步：停止旧服务并备份
```bash
sudo systemctl stop tdsql-sqlcheck
sudo cp -a /opt/tdsql-sqlcheck /opt/tdsql-sqlcheck-v1.1.0.0-backup
```

### 第 4 步：覆盖更新文件
```bash
sudo cp -r backend/ /opt/tdsql-sqlcheck/
sudo cp -r frontend/ /opt/tdsql-sqlcheck/
sudo cp requirements.txt /opt/tdsql-sqlcheck/
sudo cp VERSION /opt/tdsql-sqlcheck/
sudo cp -r deploy/ /opt/tdsql-sqlcheck/
```

### 第 5 步：更新 Python 依赖
```bash
cd /opt/tdsql-sqlcheck
sudo ./venv/bin/pip install --no-index --find-links=/tmp/dist/tdsql-sqlcheck-v1.1.0.1-linux-x86_64/wheels/ -r requirements.txt
```

### 第 6 步：启动服务
```bash
sudo systemctl start tdsql-sqlcheck
sudo systemctl status tdsql-sqlcheck
```
