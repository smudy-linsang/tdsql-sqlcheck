## 二、本次更新内容（v1.0.3.3 → v1.0.3.4）

### 缺陷修复（2项）

| 编号 | 模块 | 问题描述 | 修复内容 |
|------|------|---------|---------|
| BUG-1 | 慢SQL治理 | 多SET合并时分布写入失败（Data too long 1406） | `slow_queries.set_id` 列从 VARCHAR(32) 扩展到 VARCHAR(512)，支持多SET合并场景 |
| BUG-2 | 慢SQL治理 | 报告/看板严重度统计错误（CRITICAL→ERROR） | 慢SQL体系只有 ERROR/WARNING/INFO 三个级别，修复统计逻辑 |

### 无变更项

- 数据库结构：仅 `slow_queries.set_id` 列长度变更（自动迁移）
- 配置文件（.env）：无变更，v1.0.3.3 的 .env 完全兼容
- Python依赖：无新增（仍为 38 个 wheels）
- API接口：无变更

## 四、部署步骤（严格按顺序执行）

### 第1步：SSH登录目标服务器

```bash
ssh root@10.243.16.238
```

### 第2步：进入发布包目录并校验

```bash
cd /tmp/dist
ls -lh tdsql-sqlcheck-v1.0.3.4-linux-x86_64.tar.gz*
sha256sum -c tdsql-sqlcheck-v1.0.3.4-linux-x86_64.tar.gz.sha256
```

**期望输出**: `tdsql-sqlcheck-v1.0.3.4-linux-x86_64.tar.gz: OK`

如果不是OK，**停止操作**，报告校验失败。

### 第3步：解压发布包

```bash
tar -xzf tdsql-sqlcheck-v1.0.3.4-linux-x86_64.tar.gz
cd tdsql-sqlcheck-v1.0.3.4-linux-x86_64
ls -la
```

确认目录内有 `backend/`、`frontend/`、`deploy/`、`wheels/`、`requirements.txt`、`VERSION` 等文件。

### 第4步：停止当前服务

```bash
sudo systemctl stop tdsql-sqlcheck
sudo systemctl status tdsql-sqlcheck
```

确认服务已停止（Active: inactive）。

### 第5步：备份当前版本

```bash
sudo cp -a /opt/tdsql-sqlcheck /opt/tdsql-sqlcheck-v1.0.3.3-backup
ls -la /opt/tdsql-sqlcheck-v1.0.3.3-backup/
```

确认备份目录存在且内容完整。

### 第6步：覆盖更新代码（保留 .env 和 data/）

```bash
# 覆盖后端代码
sudo cp -r /tmp/dist/tdsql-sqlcheck-v1.0.3.4-linux-x86_64/backend/ /opt/tdsql-sqlcheck/

# 覆盖前端代码
sudo cp -r /tmp/dist/tdsql-sqlcheck-v1.0.3.4-linux-x86_64/frontend/ /opt/tdsql-sqlcheck/

# 覆盖requirements.txt
sudo cp /tmp/dist/tdsql-sqlcheck-v1.0.3.4-linux-x86_64/requirements.txt /opt/tdsql-sqlcheck/

# 覆盖VERSION文件
sudo cp /tmp/dist/tdsql-sqlcheck-v1.0.3.4-linux-x86_64/VERSION /opt/tdsql-sqlcheck/

# 覆盖部署脚本（可选，供后续运维使用）
sudo cp -r /tmp/dist/tdsql-sqlcheck-v1.0.3.4-linux-x86_64/deploy/ /opt/tdsql-sqlcheck/
```

**重要**：不要覆盖 `.env` 文件和 `data/` 目录，保留原有配置和数据。

### 第7步：安装/更新Python依赖（离线）

```bash
cd /opt/tdsql-sqlcheck
sudo /opt/tdsql-sqlcheck/venv/bin/pip install --no-index --find-links=/tmp/dist/tdsql-sqlcheck-v1.0.3.4-linux-x86_64/wheels/ -r requirements.txt
```

**期望输出**: 所有依赖 `Successfully installed` 或 `already satisfied`。

### 第8步：启动服务

```bash
sudo systemctl start tdsql-sqlcheck
sleep 5
sudo systemctl status tdsql-sqlcheck
```

确认服务 Active: active (running)。

### 第9步：执行部署验证脚本

```bash
cd /opt/tdsql-sqlcheck
sudo bash deploy/verify_deploy.sh
```

**期望输出**: 全部 PASS，0 FAIL。

或手动验证：

```bash
# 健康检查（版本号应为1.0.3.4）
curl -s http://127.0.0.1:8000/health | python3 -m json.tool

# 前端可访问
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:8000/

# 静态资源
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:8000/static/js/app.js
```

## 七、重要注意事项

1. **不要修改 `.env` 文件**：v1.0.3.3 的配置文件完全兼容 v1.0.3.4
2. **数据库自动迁移**：`slow_queries.set_id` 列长度变更会在服务启动时自动执行
3. **保留备份**：`/opt/tdsql-sqlcheck-v1.0.3.3-backup` 不要删除，用于回滚
4. **分布式实例配置**：如需使用慢SQL跨SET合并功能，请在实例配置中填写SET列表（从赤兔获取，逗号分隔）
