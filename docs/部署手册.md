## 2. 快速部署（Docker）

### 2.1 使用 Docker Compose（推荐）

**步骤一：获取项目代码**

```bash
git clone <repository-url>
cd TDSQL-SQLCheck
```

**步骤二：创建 `docker-compose.yml`**

```yaml
version: "3.8"

services:
  tdsql-sqlcheck:
    build: .
    container_name: tdsql-sqlcheck
    ports:
      - "8000:8000"
    environment:
      # TDSQL 连接配置（按实际情况修改）
      - TDSQL_HOST=10.0.0.1
      - TDSQL_PORT=3306
      - TDSQL_USER=root
      - TDSQL_PASSWORD=your_password
      - TDSQL_DATABASE=mydb
      # 定时任务配置
      - SCHEDULER_ENABLED=true
      - SCHEDULER_CRON_HOUR=2
      - SCHEDULER_CRON_MINUTE=0
      # GitLab Webhook密钥（可选）
      - GITLAB_WEBHOOK_SECRET=your_secret_token
    volumes:
      # 持久化SQLite数据库和报告文件
      - ./data:/app/data
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 10s
```

**步骤三：创建 `Dockerfile`**

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# 安装系统依赖
RUN apt-get update && apt-get install -y --no-install-recommends \
    curl \
    && rm -rf /var/lib/apt/lists/*

# 安装 Python 依赖
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制项目代码
COPY . .

# 创建数据目录
RUN mkdir -p /app/data/reports

# 暴露端口
EXPOSE 8000

# 启动命令
CMD ["python", "-m", "uvicorn", "backend.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

**步骤四：启动服务**

```bash
docker compose up -d
```

**步骤五：验证部署**

```bash
# 查看容器状态
docker compose ps

# 检查健康状态
curl http://localhost:8000/health

# 查看日志
docker compose logs -f
```

### 2.2 使用单个 Docker 命令

```bash
# 构建镜像
docker build -t tdsql-sqlcheck:0.4.0 .

# 运行容器
docker run -d \
  --name tdsql-sqlcheck \
  -p 8000:8000 \
  -e TDSQL_HOST=10.0.0.1 \
  -e TDSQL_PORT=3306 \
  -e TDSQL_USER=root \
  -e TDSQL_PASSWORD=your_password \
  -e TDSQL_DATABASE=mydb \
  -e SCHEDULER_ENABLED=true \
  -v $(pwd)/data:/app/data \
  --restart unless-stopped \
  tdsql-sqlcheck:0.4.0
```

## 3. 传统部署（pip）

### 3.1 安装步骤

**步骤一：获取项目代码**

```bash
git clone <repository-url>
cd TDSQL-SQLCheck
```

**步骤二：创建虚拟环境（推荐）**

```bash
# 创建虚拟环境
python -m venv .venv

# 激活虚拟环境
# Linux / macOS
source .venv/bin/activate
# Windows
.venv\Scripts\activate
```

**步骤三：安装依赖**

```bash
pip install -r requirements.txt
```

依赖清单：

| 包名       | 版本要求    | 用途              |
|-----------|-----------|-------------------|
| fastapi    | ≥0.115.0  | Web框架            |
| uvicorn    | ≥0.34.0   | ASGI服务器         |
| sqlglot    | ≥26.0.0   | SQL解析引擎         |
| pydantic   | ≥2.0.0    | 数据验证            |
| pymysql    | ≥1.1.0    | TDSQL/MySQL连接    |
| apscheduler| ≥3.10.0   | 定时任务调度         |
| reportlab  | ≥4.0.0    | PDF报告生成         |

> **可选依赖**: 如需PDF报告导出功能，`reportlab` 是必须的。如不使用TDSQL连接功能，`pymysql` 可不安装。

**步骤四：启动服务**

```bash
# 生产环境（推荐）
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000

# 开发环境（支持热重载）
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload
```

**步骤五：访问系统**

- **前端界面**: http://localhost:8000
- **API文档（Swagger）**: http://localhost:8000/docs
- **健康检查**: http://localhost:8000/health

### 3.2 使用 systemd 管理服务（Linux）

创建 `/etc/systemd/system/tdsql-sqlcheck.service`：

```ini
[Unit]
Description=TDSQL SQL审核工具
After=network.target

[Service]
Type=simple
User=app
Group=app
WorkingDirectory=/opt/TDSQL-SQLCheck
Environment="PATH=/opt/TDSQL-SQLCheck/.venv/bin"
EnvironmentFile=/opt/TDSQL-SQLCheck/.env
ExecStart=/opt/TDSQL-SQLCheck/.venv/bin/python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动并设置开机自启：

```bash
sudo systemctl daemon-reload
sudo systemctl enable tdsql-sqlcheck
sudo systemctl start tdsql-sqlcheck
sudo systemctl status tdsql-sqlcheck
```

## 4. 环境变量配置

所有配置均可通过环境变量设置。创建 `.env` 文件（位于项目根目录）：

```bash
# ===== TDSQL 连接配置 =====
TDSQL_HOST=10.0.0.1
TDSQL_PORT=3306
TDSQL_USER=root
TDSQL_PASSWORD=your_password_here
TDSQL_DATABASE=mydb
TDSQL_CHARSET=utf8mb4
TDSQL_CONNECT_TIMEOUT=5
TDSQL_READ_TIMEOUT=10

# ===== 定时任务配置 =====
SCHEDULER_ENABLED=true
SCHEDULER_CRON_HOUR=2
SCHEDULER_CRON_MINUTE=0
SCHEDULER_SLOW_QUERY_LIMIT=100
SCHEDULER_SLOW_QUERY_MIN_TIME=1.0

# ===== GitLab 集成 =====
GITLAB_WEBHOOK_SECRET=your_secret_token

# ===== 审核报告 =====
REPORT_OUTPUT_DIR=./data/reports
```

### 环境变量说明

| 变量名                       | 默认值    | 说明                                     |
|-----------------------------|----------|------------------------------------------|
| `TDSQL_HOST`                | (空)      | TDSQL实例地址                             |
| `TDSQL_PORT`                | 3306     | TDSQL端口                                |
| `TDSQL_USER`                | (空)      | 连接用户名                                |
| `TDSQL_PASSWORD`            | (空)      | 连接密码                                 |
| `TDSQL_DATABASE`            | (空)      | 默认数据库                                |
| `TDSQL_CHARSET`             | utf8mb4  | 连接字符集                                |
| `TDSQL_CONNECT_TIMEOUT`     | 5        | 连接超时（秒）                             |
| `TDSQL_READ_TIMEOUT`        | 10       | 读取超时（秒）                             |
| `SCHEDULER_ENABLED`         | false    | 是否启用定时任务                            |
| `SCHEDULER_CRON_HOUR`       | 2        | 定时任务执行小时（0-23）                     |
| `SCHEDULER_CRON_MINUTE`     | 0        | 定时任务执行分钟（0-59）                     |
| `SCHEDULER_SLOW_QUERY_LIMIT`| 100      | 每次拉取慢SQL最大条数                       |
| `SCHEDULER_SLOW_QUERY_MIN_TIME` | 1.0  | 慢SQL最小耗时阈值（秒）                     |
| `GITLAB_WEBHOOK_SECRET`     | (空)      | GitLab Webhook密钥                        |
| `REPORT_OUTPUT_DIR`         | ./data/reports | PDF报告输出目录                      |

## 5. TDSQL连接配置

系统支持两种TDSQL连接方式：环境变量和JSON配置文件。

### 5.1 优先级规则

```
环境变量 > 配置文件(config/tdsql.json) > 默认值
```

### 5.2 JSON配置文件

创建 `config/tdsql.json`（位于项目根目录下的 `config` 目录）：

```json
{
    "host": "10.0.0.1",
    "port": 3306,
    "user": "readonly_user",
    "password": "your_password",
    "database": "mydb",
    "charset": "utf8mb4",
    "connect_timeout": 5,
    "read_timeout": 10
}
```

> **安全建议**: 建议创建只读账号用于SQL审核和慢查询抓取，避免使用高权限账号。

### 5.3 通过API连接

也可以通过API动态连接TDSQL实例（无需预配置）：

```bash
# 方式一：通过API参数连接
curl -X POST http://localhost:8000/api/v1/tdsql/connect \
  -H "Content-Type: application/json" \
  -d '{
    "host": "10.0.0.1",
    "port": 3306,
    "user": "root",
    "password": "xxx",
    "database": "mydb"
  }'

# 方式二：使用配置文件/环境变量连接
curl -X POST http://localhost:8000/api/v1/tdsql/connect-from-config

# 测试连接
curl "http://localhost:8000/api/v1/tdsql/test-connection?host=10.0.0.1&port=3306&user=root&password=xxx"
```

## 6. GitLab Webhook配置

### 6.1 在GitLab中配置Webhook

1. 打开GitLab项目 → **Settings** → **Webhooks**
2. **URL** 填写：
   ```
   http://<your-host>:8000/api/v1/gitlab/webhook/merge-request
   ```
3. **Secret Token** 填写（与环境变量 `GITLAB_WEBHOOK_SECRET` 一致，可选）
4. **Trigger** 勾选：`Merge request events`
5. **SSL verification** 根据实际情况配置（内网可关闭）
6. 点击 **Add webhook** 保存

### 6.2 Webhook工作流程

```
开发者提交MR → GitLab发送Webhook → 本工具接收 → 提取SQL变更 → 执行审核 → 返回结果
```

系统会自动：
- 识别 `.sql` 和 `.xml`（MyBatis）文件中的SQL变更
- 提取Git Diff中的新增SQL语句
- 执行22条规则审核
- 生成格式化的审核报告

### 6.3 支持的文件类型

| 文件类型 | 说明                           |
|---------|-------------------------------|
| `.sql`  | 纯SQL脚本文件                   |
| `.xml`  | MyBatis Mapper XML文件          |
| `*Mapper.xml` | MyBatis Mapper标准命名     |

### 6.4 Webhook安全验证

如需启用Token验证：

```bash
# 在 .env 或环境变量中设置
GITLAB_WEBHOOK_SECRET=your_secret_token_here
```

系统使用 HMAC 恒定时间比较算法验证 `X-Gitlab-Token` 请求头，防止时序攻击。

## 7. 定时任务配置

### 7.1 启用定时任务

定时任务默认关闭。通过环境变量启用：

```bash
SCHEDULER_ENABLED=true
```

### 7.2 调度配置

| 环境变量                       | 默认值 | 说明                       |
|------------------------------|-------|---------------------------|
| `SCHEDULER_ENABLED`          | false | 是否启用定时任务              |
| `SCHEDULER_CRON_HOUR`        | 2     | 执行小时（0-23，默认凌晨2点）  |
| `SCHEDULER_CRON_MINUTE`      | 0     | 执行分钟（0-59）              |
| `SCHEDULER_SLOW_QUERY_LIMIT` | 100   | 每次拉取最大条数              |
| `SCHEDULER_SLOW_QUERY_MIN_TIME` | 1.0 | 最小耗时阈值（秒）            |

### 7.3 内置定时任务

| 任务名称           | 执行频率          | 说明                              |
|-------------------|------------------|----------------------------------|
| 定时拉取慢查询日志   | 每天指定时间       | 从TDSQL performance_schema + slow_log 拉取并分析 |
| TDSQL连接健康检查   | 每小时整点         | 检查TDSQL连接可用性，异常时记录日志    |

### 7.4 手动触发

```bash
# 手动触发一次慢日志拉取
curl -X POST http://localhost:8000/api/v1/tdsql/scheduler/trigger

# 查看定时任务状态
curl http://localhost:8000/api/v1/tdsql/scheduler/status
```

### 7.5 调度器特性

- **时区**: 固定为 `Asia/Shanghai`
- **任务合并**: 错过的任务合并为一次执行（`coalesce=True`）
- **单实例**: 同一任务最多一个运行实例（`max_instances=1`）
- **容错**: 错过1小时内仍会执行（`misfire_grace_time=3600`）

## 9. 常见问题

### Q1: 启动时报 `ModuleNotFoundError: No module named 'backend'`

**原因**: 未在项目根目录执行启动命令。

**解决**: 确保在 `TDSQL-SQLCheck` 目录下执行：

```bash
cd TDSQL-SQLCheck
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

### Q2: TDSQL连接失败

**排查步骤**:

```bash
# 1. 测试网络连通性
ping <TDSQL_HOST>
telnet <TDSQL_HOST> 3306

# 2. 通过API测试连接
curl "http://localhost:8000/api/v1/tdsql/test-connection?host=<HOST>&user=<USER>&password=<PWD>"

# 3. 检查pymysql是否安装
pip show pymysql
```

**常见原因**:
- 防火墙未开放3306端口
- 用户名或密码错误
- TDSQL实例未开启远程访问
- `pymysql` 未安装（`pip install pymysql`）

### Q3: PDF报告导出报错 `reportlab 未安装`

**解决**:

```bash
pip install reportlab>=4.0.0
```

### Q4: 定时任务未执行

**排查**:

1. 确认环境变量 `SCHEDULER_ENABLED=true`
2. 确认TDSQL连接参数已配置（`TDSQL_HOST` 和 `TDSQL_USER` 非空）
3. 查看调度器状态：`GET /api/v1/tdsql/scheduler/status`
4. 检查应用日志中是否有 `[定时任务]` 相关输出

### Q5: GitLab Webhook未触发审核

**排查**:

1. 确认GitLab Webhook URL正确且网络可达
2. 确认Trigger勾选了 `Merge request events`
3. 确认MR中包含 `.sql` 或 `.xml` 文件变更
4. 如配置了Secret Token，确认与环境变量 `GITLAB_WEBHOOK_SECRET` 一致
5. 在GitLab Webhook设置中查看Recent deliveries的请求/响应详情

### Q6: MyBatis XML文件审核结果为空

**原因**: XML中无 `<select>`、`<insert>`、`<update>`、`<delete>` 标签，或标签内容为空。

**解决**: 确保XML中包含标准MyBatis SQL标签且有实际SQL内容。

### Q7: SQLite数据库文件在哪里？

数据库文件位于项目根目录下的 `data/` 目录：

```
TDSQL-SQLCheck/
└── data/
    ├── tdsql_sqlcheck.db    # SQLite主数据库
    └── reports/             # PDF报告输出目录
```

### Q8: 如何升级版本？

```bash
# 1. 拉取最新代码
git pull origin main

# 2. 更新依赖
pip install -r requirements.txt --upgrade

# 3. 重启服务
# systemd方式
sudo systemctl restart tdsql-sqlcheck

# Docker方式

```

### Q9: 端口被占用如何修改？

```bash

```

Docker方式修改 `docker-compose.yml` 中的端口映射：

```yaml

  - "9000:8000"
```

### Q10: 日志在哪里查看？

- **控制台输出**: 启动时的日志直接输出到终端
- **systemd日志**: `journalctl -u tdsql-sqlcheck -f`
- **Docker日志**: `docker compose logs -f`

日志格式：`%(asctime)s [%(name)s] %(levelname)s: %(message)s`
