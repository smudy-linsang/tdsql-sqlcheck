# TDSQL 数据库 SQL 审核工具 V1.2.0.5 官方升级与部署说明书

> **生产基线版本**: **v1.1.0.1** / **v1.2.0.4**  
> **目标升级版本**: **v1.2.0.5 (慢SQL数据源升级与应急诊断图形化版)**  
> **升级路径**: **v1.1.0.1 / v1.2.0.4 ➔ v1.2.0.5 (平滑无缝升级)**  
> **发布日期**: 2026-07-23  
> **目标适用环境**: 银河麒麟 Linux V10 SP3 / CentOS / RHEL / Ubuntu / Windows Server 纯内网隔离环境  

---

## 一、V1.2.0.5 核心变更

1. **慢SQL治理默认切为全网慢SQL(monitordb,推荐)**：
   - 界面新建扫描任务默认选中 `monitordb`；
   - API 校验模型 `SlowQueryFetchRequest` 与 `ScanScheduleRequest` 默认使用 `monitordb`。

2. **应急诊断界面全面图形化与表格化重构**：
   - 移除了原始 JSON pre 代码块展示；
   - 重构为实例健康描述卡片、活跃会话表格、未提交大事务表格、锁等待表格、长 SQL 表格以及 InnoDB 死锁 Alert 警示面板。

3. **继承 V1.2.0.4 高并发与清洗基线**：
   - 122 个 DB 路由 `async def` ➔ `def` 转换；
   - 数据库连接池扩容至 60 兼按需探活；
   - 历史 `10.0.8.21 ~ 23` 脏数据清理。

---

## 二、部署与升级指令

```bash
# 1. 停止旧服务与备份
sudo systemctl stop tdsql-sqlcheck || killall python || true
sudo cp -a /opt/tdsql-sqlcheck /opt/tdsql-sqlcheck-backup-$(date +%Y%m%d)

# 2. 解压 v1.2.0.5 发布包
cd /opt/
sudo tar -xzf /tmp/tdsql-sqlcheck-v1.2.0.5-linux-x86_64.tar.gz -C /opt/
sudo chown -R root:root /opt/tdsql-sqlcheck

# 3. 激活虚拟环境并安装依赖
cd /opt/tdsql-sqlcheck
source venv/bin/activate || python3 -m venv venv && source venv/bin/activate
if [ -d "wheels" ]; then
    pip install --no-index --find-links=wheels -r requirements.txt || true
else
    pip install -r requirements.txt || true
fi

# 4. 数据清洗与 DB 初始化
python -c "import sys, os; sys.path.insert(0, '.'); from backend.services.database import ensure_db, _get_connection; ensure_db(); conn = _get_connection(); cur = conn.execute(\"DELETE FROM server_daily_inspection WHERE ip IN ('10.0.8.21', '10.0.8.22', '10.0.8.23')\"); conn.commit(); print('[Data Clean] 历史假 IP 清洗完成')"

# 5. 后台拉起服务 (4 Workers)
nohup python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 --workers 4 > backend_restart.log 2>&1 &

# 6. 健康检查
curl -s http://localhost:8000/health
```
**期望标准输出**：`{"status":"ok","version":"1.2.0.5"}`
