## 二、本次更新内容 (v1.0.4.2 → v1.0.4.3)

本次更新解决了在内网真实环境使用 `monitordb` 扫描慢日志保存入库时发生的字段长度溢出报错：

### 1. 扩宽 client_host 和 client_user 字段为 TEXT 类型
* **问题现象**：在真实大规模内网环境中，一个 slow SQL 可能会被非常多不同的应用服务器/主机连接调用。`monitordb` 做数据源时，会将这些客户端 IP 通过 `GROUP_CONCAT` 合并在一起返回。当汇总的 IP 列表长度超过原本定义的 128 字符时，保存入库会触发 MySQL 错误：
  `(1406, "Data too long for column 'client_host' at row 1")`。
* **优化内容**：
  - 更新元数据表定义：在 [database.py](file:///c:/TDSQL_SQLCHECK/TDSQL-SQLCheck/backend/services/database.py) 中，将 `slow_queries` 表中的 `client_user` 和 `client_host` 两个字段更改为 `TEXT` 类型。
  - 自动执行数据迁移：系统在新版本启动时，会通过 `Lifespan` 钩子自动执行 `ALTER TABLE slow_queries MODIFY COLUMN ... TEXT` 的幂等迁移，将已存在的存量表对应列宽度进行自动平滑扩宽。
  - 后端防御性截断：在 [scan_service.py](file:///c:/TDSQL_SQLCHECK/TDSQL-SQLCheck/backend/services/scan_service.py) 中，增加防御性截断代码，对于极端的字段溢出限制在 `4000` 字符以内，双重确保底层数据绝对安全入库。

## 四、升级步骤（严格按顺序执行）

### 第 1 步：上传并校验发布包
将发布包 `tdsql-sqlcheck-v1.0.4.3-linux-x86_64.tar.gz` 拷贝至目标服务器的 `/tmp/dist/` 目录下：
```bash
cd /tmp/dist/
sha256sum -c tdsql-sqlcheck-v1.0.4.3-linux-x86_64.tar.gz.sha256
```
**期望输出**：`tdsql-sqlcheck-v1.0.4.3-linux-x86_64.tar.gz: OK`
*若校验失败，请立即停止部署并重新传输文件。*

### 第 2 步：解压发布包
```bash
tar -xzf tdsql-sqlcheck-v1.0.4.3-linux-x86_64.tar.gz
cd tdsql-sqlcheck-v1.0.4.3-linux-x86_64
ls -la
```
确认存在 `backend/`、`frontend/`、`deploy/`、`wheels/`、`requirements.txt`、`VERSION` 等文件和目录。

### 第 3 步：停止运行中的旧服务
```bash
sudo systemctl stop tdsql-sqlcheck
sudo systemctl status tdsql-sqlcheck
```
确保服务已停止且状态为 `inactive (dead)`。

### 第 4 步：备份旧版本
```bash
sudo cp -a /opt/tdsql-sqlcheck /opt/tdsql-sqlcheck-v1.0.4.2-backup
ls -la /opt/tdsql-sqlcheck-v1.0.4.2-backup/
```
确保备份完整，便于在升级失败时秒级回滚。

### 第 5 步：覆盖更新文件 (保留配置与临时数据)
```bash
# 复制后端核心代码
sudo cp -r backend/ /opt/tdsql-sqlcheck/

# 复制前端核心静态包
sudo cp -r frontend/ /opt/tdsql-sqlcheck/

# 复制依赖说明文件
sudo cp requirements.txt /opt/tdsql-sqlcheck/

# 复制版本标记文件
sudo cp VERSION /opt/tdsql-sqlcheck/

# 复制新版运维与验证脚本
sudo cp -r deploy/ /opt/tdsql-sqlcheck/
```
> [!IMPORTANT]
> **请勿覆盖** `/opt/tdsql-sqlcheck/.env` 文件以及 `/opt/tdsql-sqlcheck/data/` 目录，以保持原有的数据库连接设置与口令！

### 第 6 步：升级 Python 离线依赖包 (利用内置 wheels)
```bash
cd /opt/tdsql-sqlcheck
sudo ./venv/bin/pip install --no-index --find-links=/tmp/dist/tdsql-sqlcheck-v1.0.4.3-linux-x86_64/wheels/ -r requirements.txt
```
**期望输出**：`Successfully installed` 或 `already satisfied`。

### 第 7 步：启动服务并自动执行元数据迁移
```bash
sudo systemctl start tdsql-sqlcheck
sleep 5
sudo systemctl status tdsql-sqlcheck
```
*启动过程中，应用会自动连接元数据库，并将 slow_queries 表的 client_host 与 client_user 字段修改为 TEXT 类型。*

### 第 8 步：执行部署验证脚本
```bash
cd /opt/tdsql-sqlcheck
sudo bash deploy/verify_deploy.sh
```
**期望输出**：
```text

  [PASS] 健康检查 /health
```
