# v1.1.0 · TDSQL 原厂诊断能力吸收 —— 交付说明

> 版本：v1.1.0（原 v1.0.3.4 之上的大版本能力升级）
> 目标：在**完整保留现有功能**（全量回归 885→**917 passed / 55 skipped / 0 failed**）的前提下，吸收 TDSQL 原厂工具集 `tdsql-toolkit` 中"我方没有"的诊断能力。
> 说明：本次为**后端能力 + API + 测试 + 前端入口**的完整交付；每项均在真库/mock 库自测通过并推 main。

## 一、已交付能力（G1–G9）

| 编号 | 能力 | 数据源 | 后端服务 | API 前缀 | 测试文件 |
|---|---|---|---|---|---|
| **G1** | monitordb 集群级慢SQL数据源 | monitordb `proxy_classes_analysis` | `tdsql_connector.get_cluster_slow_queries` | 慢SQL扫描 `source=monitordb` + `/tdsql/.../monitor-probe` | `test_monitordb_slow.py` |
| **G2** | 慢SQL 十列增强诊断 | 业务库(EXPLAIN/IS/mysql/sys) | `slow_enrich_service` | 随扫描自动增强 | `test_slow_enrich.py` |
| **G3** | 集群深度巡检 | monitordb `m_data_cur` | `cluster_inspect_service` | `/api/v1/cluster-inspect` | `test_cluster_inspect.py` |
| **G4** | 每日巡检 + 多日趋势 | monitordb `m_data_cur` | `daily_inspect_service` | `/api/v1/daily-inspect` | `test_daily_inspect.py` |
| **G5** | 实例级索引健康审计 | 业务库 information_schema | `index_audit_service` | `/api/v1/index-audit` | `test_index_audit.py` |

**新增元数据库表**：`cluster_inspection(_issue)`、`index_audit(_finding)`、`schema_diff(_item)`、`emergency_report`、`daily_inspection`、`bigtable_history`；`tdsql_connections` 增 `monitor_*` 列；`slow_queries` 增 `rows_affected` + 十项增强列。全部幂等迁移、带默认值，**存量库零影响**。

**前端**：新增「深度诊断」菜单（`deep-diag`），内含 集群巡检 / 索引体检 / 结构比对 / 应急诊断 / SQL分析 五个 Tab，接后端 API。

**贯穿红线**：① 被诊断实例只读（EXPLAIN 仅 `EXPLAIN SELECT`，UPDATE/DELETE 转写、拒分号、门禁校验）；② 严重度经 `engine/severity_map.py` 统一为 **ERROR/WARNING/INFO**（原厂 FATAL/CRITICAL/HIGH/MEDIUM 一律映射，不透传前端）；③ 采纳评审整改（权限降级不中断主流程、锁等待表 5.7/8.0 版本自适应、防除零、时间窗走索引列 timestramp、`/*sets:allsets*/` 不支持自动降级）。
