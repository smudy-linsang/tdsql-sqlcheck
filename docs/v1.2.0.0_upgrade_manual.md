## 二、本次更新内容 (v1.1.0.1 → v1.2.0.0)

本次更新为系统的 v1.2.0.0 大版本演进，全面落地了架构工程化与用户体验升级：

1. **Schema 版本化与增量 Migration (C1)**：
   - 抽离 DDL 到 `backend/schema/vN/*.sql` 文件，启动时自动计算 SHA256 Checksum 并安全执行增量 DDL。
2. **Engine & Parser 拆分与 TDSQL 分布式检测 (C2 + E1)**：
   - 解耦 `PreParser` 与 `ASTParser`；
   - 新增 `TDSQLAuditor`，专职校验 Shard Key 缺失广播扫描、GSI 延迟风险与广播表高高危锁表防线。
3. **TDSQL Connector 专职解耦 (C3)**：
   - 拆分为 `connection_pool.py`, `metadata_fetcher.py`, `slow_query_fetcher.py`, `monitordb_client.py` 4 大模块。
4. **ToolBridge 运维工具箱与流式 NDJSON 批处理 API (C6 + E2)**：
   - 支持异步工具调度、日志控制台抽屉与大文件/多 SQL 流式分片审核。
5. **RBAC 全矩阵自动化测试 (C5)**：
   - 新增 `tests/test_v2_rbac_matrix.py` 自动化防护用例。
6. **Vite + Vue 3 渐进式双 UI 架构 (C4 + E3 + E4)**：
   - 现代化前端、Monaco Editor 智能高亮，以及 `/?ui=v1` 动态切回极简单页的离线降级机制。

## 三、升级步骤

### 第 1 步：上传并校验发布包
```bash
cd /tmp/dist/
sha256sum -c tdsql-sqlcheck-v1.2.0.0-linux-x86_64.tar.gz.sha256
```
**期望输出**：`tdsql-sqlcheck-v1.2.0.0-linux-x86_64.tar.gz: OK`

### 第 2 步：停止服务与备份
```bash
sudo systemctl stop tdsql-sqlcheck
sudo cp -a /opt/tdsql-sqlcheck /opt/tdsql-sqlcheck-v1.1.0.1-backup
```

### 第 3 步：解压并覆盖更新
```bash
tar -xzf tdsql-sqlcheck-v1.2.0.0-linux-x86_64.tar.gz
cd tdsql-sqlcheck-v1.2.0.0-linux-x86_64
sudo cp -r backend/ /opt/tdsql-sqlcheck/
sudo cp -r frontend/ /opt/tdsql-sqlcheck/
sudo cp requirements.txt /opt/tdsql-sqlcheck/
sudo cp VERSION /opt/tdsql-sqlcheck/
sudo cp -r deploy/ /opt/tdsql-sqlcheck/
```

### 第 4 步：启动服务并验证
```bash
sudo systemctl start tdsql-sqlcheck
curl -s http://127.0.0.1:8000/health
```
**期望输出**：`{"status":"ok","version":"1.2.0.0"}`
