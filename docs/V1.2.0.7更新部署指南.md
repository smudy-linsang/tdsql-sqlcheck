# TDSQL 数据库 SQL 审核工具 V1.2.0.7 更新部署指南

> **更新类型**: P0 级权限体验优化 + 实例管理修复 + 内网交付标准重构定版  
> **版本号**: `v1.2.0.7`  
> **发布日期**: 2026-07-24  
> **适用生产环境**: 银河麒麟高级服务器版 V10 SP3 / CentOS / RHEL (x86_64, 纯内网隔离)

## 一、本次 V1.2.0.7 核心修复与交付重构要素

### 1. 彻底消除多角色登录/首次登录强制改口令时的“霸屏无权弹窗” (P0)
- **拦截器优化**：移除了前端网络拦截器 `apiFetch` 中的 403 强行全局 Warning 弹窗提示。只读数据请求被隔离时进行静默处理，不再干扰用户。
- **动态按需加载**：重构 `loadAll()` 逻辑，优先获取 `visibleMenus`（账号可访问菜单集合），仅对账号有权访问的模块进行 API 数据请求。
- **智能首屏自动引导**：受限角色（如开发人员 `developer`）登录后，系统自动从无权页面重定向至拥有权限的第一个有效工作台页面（如 `sql-audit`）。
- **口令修改挂起保护**：首次登录强制修改口令时，背景数据加载自动挂起，防止背景并发请求。

### 2. 修复实例管理与体检下拉框数据加载逻辑 (P0)
- 修正了 `app.js` 中实例管理权限 Key (`instances`) 的匹配逻辑，恢复登录及页面切换时 `loadSavedConnections()` 的自动调用。数据库中添加的 TDSQL 实例列表恢复正常渲染展示。

### 3. 内网打包与离线部署依赖全量优化
- **wheels 依赖纠正**：`requirements.txt` 中将 `uvicorn[standard]` 改为 `uvicorn>=0.34.0`，从源头解决内网安装时因为缺少 `uvloop` 导致 `pip install` 退出报错的问题。
- **便携 Python 自动兼容**：`preflight_check.sh` 与 `install.sh` 自动感知麒麟系统 `/opt/python311/python/bin` 便携 Python 路径，部署时无需运维人员手动导出环境变量。
- **提供一键回滚脚本**：新增 `deploy/rollback.sh` 脚本并打入部署包。
- **版本号自动化同步**：全局 `VERSION`、`backend/config.py`、`deploy/install.sh` 及前端 `index.html` 的版本标识均自动化一致同步为 `1.2.0.7`。

## 二、部署环境 `.env` 配置文件完整字段说明示例

部署前请复制模板并填写数据库等配置 (`cp deploy/env.template deploy/.env`)：

```ini
# ── 系统元数据库（TDSQL 集中式实例，MySQL 协议）【必填】 ──
SQLCHECK_DB_HOST=10.243.16.238
SQLCHECK_DB_PORT=3306
SQLCHECK_DB_USER=sqlcheck_app
SQLCHECK_DB_PASSWORD=YourPassword123!
SQLCHECK_DB_NAME=tdsql_sqlcheck
SQLCHECK_DB_POOL_SIZE=10

# ── 认证与安全 ──
AUTH_ENABLED=true
# HMAC 签名密钥（升级场景请保持与上一版一致；留空则由 install.sh 首次自动生成并追加）
AUTH_SECRET_KEY=
# admin 初始口令（仅首次建库创建 admin 用户时有效，首登强制修改）【必填】
ADMIN_INITIAL_PASSWORD=Admin@123
AUTH_TOKEN_TTL_HOURS=8
AUTH_MAX_LOGIN_FAILURES=5
AUTH_LOCK_MINUTES=15
DATA_MASKING_ENABLED=true
DOCS_PUBLIC=false

# ── 调度与可观测性 ──
SCHEDULER_ENABLED=true
METRICS_ENABLED=true
REPORT_OUTPUT_DIR=/opt/tdsql-sqlcheck/reports
```

### 字段含义清单

| 配置项 | 属性 | 默认/示例 | 说明 |
|---|---|---|---|
| `SQLCHECK_DB_HOST` | 必填 | `10.243.16.238` | 元数据库 TDSQL IP/域名 |
| `SQLCHECK_DB_PORT` | 必填 | `3306` | 元数据库端口 |
| `SQLCHECK_DB_USER` | 必填 | `sqlcheck_app` | 元数据库用户名 |
| `SQLCHECK_DB_PASSWORD` | 必填 | `***` | 元数据库密码 |
| `ADMIN_INITIAL_PASSWORD` | 必填 | `Admin@123` | 管理员初始密码 |
| `AUTH_SECRET_KEY` | 可选 | (留空自动生成) | 升级时保留旧 Key 避免用户登录掉线 |

## 三、部署与升级一键操作命令 (内网部署智能体适用)

```bash
# 1. 校验离线发布包 SHA256
cd /tmp
sha256sum -c tdsql-sqlcheck-v1.2.0.7-linux-x86_64.tar.gz.sha256

# 2. 解压安装包
sudo tar -xzf /tmp/tdsql-sqlcheck-v1.2.0.7-linux-x86_64.tar.gz -C /opt/
cd /opt/tdsql-sqlcheck-v1.2.0.7-linux-x86_64

# 3. 复制配置并运行一键安装
cp deploy/env.template deploy/.env
# 根据需要修改 deploy/.env 后执行：
sudo ./deploy/install.sh
```
