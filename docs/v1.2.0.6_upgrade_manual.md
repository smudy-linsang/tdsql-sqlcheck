## 一、本次 V1.2.0.6 核心修复与开发变更说明

### 1. P0 级修复：消除多角色登录/首次登录强制改口令时的“霸屏无权弹窗”
- **根本原因**：之前网络底层 `apiFetch` 拦截器对所有 403 (Forbidden) 响应统一盲目调用了 `ElMessage.warning('当前角色无权执行该操作')` Toast 提示。当开发、审计等受限角色登录后，页面背景一次性发起的 5~10 个只读数据请求被 403 过滤，导致前端屏幕中央瞬间连弹 10 条黄色消息框，极其影响体验。
- **重构解决**：
  - 彻底去除了 `apiFetch` 中的 403 全局硬弹框。
  - 数据加载逻辑重构为基于 `visibleMenus`（角色可访问菜单集合）的动态按需加载，仅针对当前账户拥有的模块进行后台 API 请求。
  - 增加首屏智能路由重定向：受限角色登录后，系统自动跳转至其拥有的第一个有效工作台页面（如开发人员直达 `sql-audit`）。
  - 强制修改密码期间挂起背景数据拉取，修改完口令前不触发无关请求。

### 2. UI 界面版本统一
- 全站及登录页底部版本标识已同步更新为 `V1.2.0.6 · Design by Linsang`。
- 系统管理 > 系统信息中的 API 响应同步为 `1.2.0.6`。

## 二、部署与升级操作步骤 (内网部署智能体适用)

### 1. 拷贝并校验升级包
将 `dist/tdsql-sqlcheck-v1.2.0.6-linux-x86_64.tar.gz` 及其 SHA256 校验文件拷贝至目标服务器 `/tmp` 目录：

```bash
cd /tmp
sha256sum -c tdsql-sqlcheck-v1.2.0.6-linux-x86_64.tar.gz.sha256
```

### 2. 解压 v1.2.0.6 最新发布包
```bash
sudo tar -xzf /tmp/tdsql-sqlcheck-v1.2.0.6-linux-x86_64.tar.gz -C /opt/
cd /opt/tdsql-sqlcheck-v1.2.0.6-linux-x86_64
```

### 3. 执行安装与重启
```bash
cp deploy/env.template deploy/.env
sudo ./deploy/install.sh
```
