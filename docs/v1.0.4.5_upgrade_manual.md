## 二、本次更新内容 (v1.0.4.4 → v1.0.4.5)

本次更新解决了一个严重的凭据覆盖问题，该问题会导致编辑保存实例连接后，后续扫描或任务运行因“密码空”而触发 1045 拒绝访问报错：

### 1. 修复编辑保存连接时密码被置空覆盖的 Bug
* **问题现象**：在前端点击“编辑实例连接”并保存后，再次发起慢 SQL 扫描任务时，系统可能会抛出类似以下拒绝登录的报错：
  `(1045, "Proxy ERROR: Access denied for user 'tdsql_check_user'@'xxx' (using password: 'YES'): username or password error")`。
* **根本原因**：为了安全起见，前端的编辑抽屉里对于密码输入框采取了隐式置空处理（防窃听）。当用户未手动修改密码直接点击“保存/更新”时，前端会将空字符串 `""` 发送至后端更新接口。后端接口收到后，直接将这个空字串进行加密并写入数据库，导致数据库中原有的正确密码被**置空抹除**。随后的任务读取到的解密密码变为空字串，从而引发 1045 错误。
* **优化内容**：
  - 更新密码保存机制：在 [connection_registry.py](file:///c:/TDSQL_SQLCHECK/TDSQL-SQLCheck/backend/services/connection_registry.py) 中重构了 `save_connection` 逻辑。
  - 增量密码更新：当更新已有连接配置时，如果用户没有输入新密码（即收到的 `password` 或 `monitor_password` 为空），系统会自动加载数据库原有的已加密密码进行复用保留，防止将其置空抹除。只有当用户明确重新输入了非空的新密码时，才会进行覆盖加密写入。

## 四、升级步骤（严格按顺序执行）

### 第 1 步：上传并校验发布包
将发布包 `tdsql-sqlcheck-v1.0.4.5-linux-x86_64.tar.gz` 拷贝至目标服务器的 `/tmp/dist/` 目录下：
```bash
cd /tmp/dist/
sha256sum -c tdsql-sqlcheck-v1.0.4.5-linux-x86_64.tar.gz.sha256
```
**期望输出**：`tdsql-sqlcheck-v1.0.4.5-linux-x86_64.tar.gz: OK`
*若校验失败，请立即停止部署并重新传输文件。*

### 第 2 步：解压发布包
```bash
tar -xzf tdsql-sqlcheck-v1.0.4.5-linux-x86_64.tar.gz
cd tdsql-sqlcheck-v1.0.4.5-linux-x86_64
ls -la
```
确认存在 `backend/`、`frontend/`、`deploy/`、`wheels/`、`requirements.txt`、`VERSION` 等文件和目录。

### 第 3 步：停止运行中的旧服务
```bash
sudo systemctl stop tdsql-sqlcheck
sudo systemctl status tdsql-sqlcheck
```
确保服务已停止且状态为 `inactive (dead)`。

### 第 4 步：备份旧版本
```bash
sudo cp -a /opt/tdsql-sqlcheck /opt/tdsql-sqlcheck-v1.0.4.4-backup
ls -la /opt/tdsql-sqlcheck-v1.0.4.4-backup/
```
确保备份完整，便于在升级失败时秒级回滚。

### 第 5 步：覆盖更新文件 (保留配置与临时数据)
```bash
# 复制后端核心代码
sudo cp -r backend/ /opt/tdsql-sqlcheck/

# 复制前端核心静态包
sudo cp -r frontend/ /opt/tdsql-sqlcheck/

# 复制依赖说明文件
sudo cp requirements.txt /opt/tdsql-sqlcheck/

# 复制版本标记文件
sudo cp VERSION /opt/tdsql-sqlcheck/

# 复制新版运维与验证脚本
sudo cp -r deploy/ /opt/tdsql-sqlcheck/
```
> [!IMPORTANT]
> **请勿覆盖** `/opt/tdsql-sqlcheck/.env` 文件以及 `/opt/tdsql-sqlcheck/data/` 目录，以保持原有的数据库连接设置与口令！

### 第 6 步：升级 Python 离线依赖包 (利用内置 wheels)
```bash
cd /opt/tdsql-sqlcheck
sudo ./venv/bin/pip install --no-index --find-links=/tmp/dist/tdsql-sqlcheck-v1.0.4.5-linux-x86_64/wheels/ -r requirements.txt
```
**期望输出**：`Successfully installed` 或 `already satisfied`。

### 第 7 步：启动服务
```bash
sudo systemctl start tdsql-sqlcheck
sleep 5
sudo systemctl status tdsql-sqlcheck
```

### 第 8 步：执行部署验证脚本
```bash
cd /opt/tdsql-sqlcheck
sudo bash deploy/verify_deploy.sh
```
**期望输出**：
```text

  [PASS] 健康检查 /health

════ 验证结果: PASS=7 FAIL=0 ════
部署验证全部通过 ✔
```
