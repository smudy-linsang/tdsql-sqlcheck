### 6.1 第三轮复检（对 G commit `52156cf` 的复验）—— BUG-V12-03 仍未修复 ❌

G 在 `52156cf` 尝试修 BUG-V12-03，但**改错了地方、问题依旧**：
- commit 实际改动：`NOW()` → Python datetime 字符串（跨库兼容）、`http_request` 形参给默认值 `None` 并加 `hasattr` 守卫；
- **未触及真正根因**：`backend/api/toolkit.py:74` 依旧是 `getattr(http_request.state, "user", None)`——取的仍是**用户字典**（`state.user` 是 dict，`state.username` 才是字符串）。
- 实测复验（认证态 `POST /api/v1/toolkit/run`）：**仍返回 500**，后端异常同前：
  ```
  File ".../backend/api/toolkit.py", line 75, in trigger_tool_run
    run_id = tool_bridge_service.create_run_task(tool_name, conn_id, params, operator)
  File ".../backend/services/tool_bridge_service.py", line 23, in create_run_task
  TypeError: dict can not be used as parameter
  ```

**精确修复（第三次重申，就一处）**：把 `toolkit.py:74` 的 `"user"` 改成 `"username"`：
```python
# 现状（错）：operator 取到的是 dict → 入库即 500
operator = (getattr(http_request.state, "user", None) if http_request and hasattr(http_request, "state") else None) or payload.get("operator", "system")
# 应改为：
operator = (getattr(http_request.state, "username", None) if http_request and hasattr(http_request, "state") else None) or payload.get("operator", "system")
```
> 中间件 `backend/middleware.py:97/99`：`state.username` 是用户名字符串、`state.user` 是用户字典。工具箱只需用户名，取 `username` 即可。改完后 `/run` 应返回 200 且 `tool_runs.created_by` 落用户名。
