## 二、本次更新内容 (v1.0.4.6 → v1.0.4.7)

本次更新解决了实例管理页面连接测试状态未能实时反馈的浏览器缓存问题：

### 1. 修复实例连接测试后状态未刷新问题
* **问题现象**：在前端 **「实例与体检」 > 「实例管理」** 页面，人工测试点击某实例的“连接”按钮进行连接时，页面弹下了“已连接”小窗口，但“状态”列的指示灯却有概率仍然停留在灰色“未连接”字样。
* **问题根本原因**：前端发送连接请求成功后，立即调用 `loadSavedConnections()` 重新拉取连接列表（通过 `GET /api/v1/tdsql/connections`）。但在部分浏览器/代理环境下，该 `GET` 请求会由于没有缓存清除机制而直接返回浏览器缓存的旧 JSON 响应（其中 `active` 仍为 `false`），从而导致 UI 状态无法与实际连接状态保持同步。
* **修改内容**：
  - 优化前端网络请求核心方法 `apiFetch`（[frontend/static/js/app.js](file:///c:/TDSQL_SQLCHECK/TDSQL-SQLCheck/frontend/static/js/app.js)）：在发送所有 `GET` 请求时，自动在 URL 尾部拼接防缓存时间戳参数 `_t=Date.now()`。
  - 彻底规避了浏览器/中间代理对 API 状态获取接口的缓存影响，确保每次点击“连接”后，绿点“已连接”状态均能秒级同步。

## 四、升级步骤（严格按顺序执行）

### 第 1 步：上传并校验发布包
将发布包 `tdsql-sqlcheck-v1.0.4.7-linux-x86_64.tar.gz` 拷贝至目标服务器的 `/tmp/dist/` 目录下：
```bash
cd /tmp/dist/
sha256sum -c tdsql-sqlcheck-v1.0.4.7-linux-x86_64.tar.gz.sha256
```
**期望输出**：`tdsql-sqlcheck-v1.0.4.7-linux-x86_64.tar.gz: OK`
*若校验失败，请立即停止部署并重新传输文件。*

### 第 2 步：解压发布包
```bash
tar -xzf tdsql-sqlcheck-v1.0.4.7-linux-x86_64.tar.gz
cd tdsql-sqlcheck-v1.0.4.7-linux-x86_64
ls -la
```
确认存在 `backend/`、`frontend/`、`deploy/`、`wheels/`、`requirements.txt`、`VERSION` 等文件和目录。

### 第 3 步：停止运行中的旧服务
```bash
sudo systemctl stop tdsql-sqlcheck
sudo systemctl status tdsql-sqlcheck
```
确保服务已停止且状态为 `inactive (dead)`。

### 第 4 步：备份旧版本
```bash
sudo cp -a /opt/tdsql-sqlcheck /opt/tdsql-sqlcheck-v1.0.4.6-backup
ls -la /opt/tdsql-sqlcheck-v1.0.4.6-backup/
```
确保备份完整，便于在升级失败时秒级回滚。

### 第 5 步：覆盖更新文件 (保留配置与临时数据)
```bash
# 复制后端核心代码
sudo cp -r backend/ /opt/tdsql-sqlcheck/

# 复制前端核心静态包
sudo cp -r frontend/ /opt/tdsql-sqlcheck/

# 复制依赖说明文件
sudo cp requirements.txt /opt/tdsql-sqlcheck/

# 复制版本标记文件
sudo cp VERSION /opt/tdsql-sqlcheck/

# 复制新版运维与验证脚本
sudo cp -r deploy/ /opt/tdsql-sqlcheck/
```
> [!IMPORTANT]
> **请勿覆盖** `/opt/tdsql-sqlcheck/.env` 文件以及 `/opt/tdsql-sqlcheck/data/` 目录，以保持原有的数据库连接设置与口令！

### 第 6 步：升级 Python 离线依赖包 (利用内置 wheels)
```bash
cd /opt/tdsql-sqlcheck
sudo ./venv/bin/pip install --no-index --find-links=/tmp/dist/tdsql-sqlcheck-v1.0.4.7-linux-x86_64/wheels/ -r requirements.txt
```
**期望输出**：`Successfully installed` 或 `already satisfied`。

### 第 7 步：启动服务
```bash
sudo systemctl start tdsql-sqlcheck
sudo systemctl status tdsql-sqlcheck
```
确保状态为 `active (running)`，且启动日志中无报错。
