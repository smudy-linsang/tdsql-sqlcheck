# TDSQL数据库SQL审核工具 部署手册

> 文档版本：v1.0.2 ｜ 适用系统版本：v1.0.2 ｜ 更新日期：2026-07-07
> 目标环境：银河麒麟高级服务器版 V10 SP3（x86_64 / aarch64）
> 元数据库：TDSQL 集中式实例（MySQL 协议）
> 读者：部署实施人员（本手册为一键部署主文档，按章节顺序执行即可）

---

## 1. 部署架构与组件清单

```
 用户浏览器 ──HTTPS 443──▶ [Nginx 可选反代/TLS] ──HTTP 8000──▶ [tdsql-sqlcheck 应用]
                                                              │  uvicorn(2 workers) + FastAPI
                                                              │  内置前端静态资源(纯内网,零外链)
                                                              ├──MySQL协议──▶ TDSQL集中式实例(系统元数据库)
                                                              └──MySQL协议──▶ 各被纳管TDSQL实例(审核目标,界面配置)
```

### 1.1 中间件与运行时清单（本系统依赖极简，无重量级中间件）

| 组件 | 版本要求 | 必须/可选 | 来源与部署方式 |
|---|---|---|---|
| Python | **≥3.9**（推荐 3.11） | 必须 | 三选一：① 目标机已有 python3.9+；② 麒麟内网源 `yum install -y python39`；③ 发布包内置便携 Python（打包时 `--with-python`） |
| uvicorn + FastAPI 等 9 个 Python 库 | requirements.txt 锁定 | 必须 | **发布包 wheels/ 离线安装**，目标机无需外网 |
| systemd | 麒麟 V10 自带 | 必须 | 服务托管（install.sh 自动安装 unit） |
| TDSQL 集中式实例 | MySQL 5.7+/8.0 兼容协议 | 必须 | 行内已有，本手册 3.2 节给建库授权 SQL |
| Nginx | 1.20+ | **可选** | 需 TLS/443 统一入口时启用；麒麟内网源 `yum install -y nginx`，配置文件见 `deploy/nginx-sqlcheck.conf` |
| Prometheus | 任意近期版本 | 可选 | 行内已有监控平台则抓取 `/metrics` 即可，本系统无需部署 |

**明确不需要**：Redis、消息队列、Java/Node 运行时、外部对象存储、Docker（可用但非必须，本手册按裸机 systemd 部署）。

### 1.2 服务器资源建议

| 规模 | CPU | 内存 | 磁盘 | 说明 |
|---|---|---|---|---|
| 纳管 ≤100 实例 | 4C | 8G | 100G | 单机即可 |
| 纳管 100–500 实例 | 8C | 16G | 200G | 单机 + 调整 `.env` 容量参数（见 4.2） |

磁盘主要消耗：元数据库（慢SQL记录/审核历史，由数据保留策略自动清理）与 `reports/` 导出报告。

## 3. TDSQL 集中式元数据库准备（DBA 执行）

### 3.1 实例参数要求

- 字符集支持 utf8mb4（建库时显式指定，不依赖 server 级默认）；
- `max_connections` 预留：本应用 2 workers × 连接池10 + 冗余 ≈ **预留 50 连接**；
- 建议为本应用创建独立数据库，并纳入行内常规备份策略（每日全备即可，库体量小）。

### 3.2 建库与授权（复制执行，替换口令）

```sql
CREATE DATABASE IF NOT EXISTS tdsql_sqlcheck DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

CREATE USER 'sqlcheck_app'@'%' IDENTIFIED BY '<强口令>';
-- 应用启动时自动建表/增量迁移（ensure_db），因此需要 DDL 权限，范围严格限制在本库
GRANT SELECT, INSERT, UPDATE, DELETE, CREATE, ALTER, INDEX, DROP, REFERENCES
  ON tdsql_sqlcheck.* TO 'sqlcheck_app'@'%';
FLUSH PRIVILEGES;
```

> 说明：表结构由应用首次启动自动创建（20+ 张表：users/rule_sets/projects/slow_queries/audit_history/operation_logs/scan_schedules 等，全清单见《系统数据库设计说明书-v1.0.2》）。**无需手工执行建表脚本。**

### 3.3 被纳管的目标 TDSQL 实例账号（各实例 DBA 执行，上线后经界面录入）

审核目标实例只需**只读+性能视图**权限：

```sql
CREATE USER 'sqlcheck_ro'@'<应用服务器IP>' IDENTIFIED BY '<强口令>';
GRANT SELECT, PROCESS ON *.* TO 'sqlcheck_ro'@'<应用服务器IP>';
GRANT SELECT ON performance_schema.* TO 'sqlcheck_ro'@'<应用服务器IP>';
```

## 4. 发布包构建（打包机执行，一次性）

在**可访问 pip 源**的打包机（Linux，python3.9+，与目标机架构无需相同）：

```bash
git clone <本仓库> && cd tdsql-sqlcheck
# 目标机为 x86_64 麒麟:
./deploy/make_release.sh --arch x86_64
# 目标机为鲲鹏/飞腾 aarch64:
./deploy/make_release.sh --arch aarch64
# 目标机确认没有 python3.9+ 且内网源也装不了时，内置便携 Python:
./deploy/make_release.sh --arch x86_64 --with-python
```

产出 `dist/tdsql-sqlcheck-v1.0.2-linux-<arch>.tar.gz`（含代码+前端+全部离线 wheels+部署脚本+文档）及 `.sha256` 校验文件。**通过行内介质摆渡至目标服务器。**

## 5. 一键部署（目标服务器执行）

### 5.1 解压与配置

```bash
sha256sum -c tdsql-sqlcheck-v1.0.2-linux-x86_64.tar.gz.sha256     # 必须 OK
tar -xzf tdsql-sqlcheck-v1.0.2-linux-x86_64.tar.gz
cd tdsql-sqlcheck-v1.0.2-linux-x86_64
cp deploy/env.template deploy/.env
vi deploy/.env
```

`.env` **必填 4 项**（其余保持模板默认即可）：

```
SQLCHECK_DB_HOST=<TDSQL集中式实例IP>
SQLCHECK_DB_PORT=<端口>
SQLCHECK_DB_PASSWORD=<sqlcheck_app口令>
ADMIN_INITIAL_PASSWORD=<admin初始口令，≥8位含大小写数字特殊字符三类>
```

### 5.2 执行一键部署

```bash
sudo ./deploy/install.sh                    # 默认 /opt/tdsql-sqlcheck + 端口8000
# 或自定义: sudo ./deploy/install.sh --dir /app/sqlcheck --port 8080
```

脚本自动完成：**预检 →** 创建 `sqlcheck` 运行用户与目录 → 部署代码至 `releases/v1.0.2` → venv + wheels 离线装依赖 → 安装 `.env`（自动生成并固化 `AUTH_SECRET_KEY`）→ 切 `current` 软链 → 安装/启动 systemd 服务 → firewalld 放通端口 → **部署后自动验证**（9 项冒烟）。

看到 `部署成功!` 且验证 `PASS=... FAIL=0` 即部署完成。

### 5.3 （可选）启用 Nginx TLS

```bash
yum install -y nginx
cp deploy/nginx-sqlcheck.conf /etc/nginx/conf.d/sqlcheck.conf
vi /etc/nginx/conf.d/sqlcheck.conf          # 改域名与证书路径
# 收敛应用只监听回环:
sed -i 's/--host 0.0.0.0/--host 127.0.0.1/' /etc/systemd/system/tdsql-sqlcheck.service
systemctl daemon-reload && systemctl restart tdsql-sqlcheck
nginx -t && systemctl enable --now nginx
firewall-cmd --permanent --add-service=https && firewall-cmd --permanent --remove-port=8000/tcp && firewall-cmd --reload
```

## 6. 部署后验证与首次初始化

### 6.1 自动验证（install.sh 已执行，可随时重跑）

```bash
sudo bash /path/to/包/deploy/verify_deploy.sh --port 8000
```

9 项检查：health=ok、版本=1.0.2、首页、3 个静态资产、admin 登录、规则总数 119（含 oracle_compat 42）、审核命中 R080、元数据库读写、/metrics 输出。**全部 PASS 才算部署完成。**

### 6.2 首次人工初始化（上线负责人执行，约 15 分钟）

1. 浏览器访问系统 → admin 登录 → **按提示强制修改初始口令**；
2. 「系统管理→用户管理」创建 dba/开发/审计员账号（角色权限见《用户使用手册》）；
3. 「实例管理」录入首批被纳管 TDSQL 实例 → 逐一「测试连接」通过 → 设置默认实例；
4. 「平台治理→项目管理」建首批项目并绑定规则集/门禁策略（默认 strict 可用）；
5. 「慢SQL治理→扫描计划」为核心实例配置每日定时扫描；
6. 「系统管理→数据保留」确认默认保留策略（建议：慢SQL 90 天、审核历史 180 天、操作日志 365 天）。

## 7. 故障排查速查

| 现象 | 定位 | 处理 |
|---|---|---|
| install.sh 预检 FAIL | 看预检输出逐项 | 按提示处理后重跑 |
| 服务起不来 | `journalctl -u tdsql-sqlcheck -n 50` | 多为 .env 数据库配置错误；改后 `systemctl restart tdsql-sqlcheck` |
| 登录报 500 / 概览无数据 | 元数据库连不上或权限不足 | 用 3.2 节账号手工 `mysql -h... -e "SELECT 1"` 验证；核对 GRANT |
| 页面空白 | 浏览器 F12 看静态资源 404 | 确认 `current/frontend/static/` 完整；`systemctl restart` |
| 实例"测试连接"失败 | 应用服务器→目标实例网络 | `telnet <实例IP> <端口>`；核对 3.3 节账号 |
| wheels 安装报错 "not supported on this platform" | 架构不匹配 | 用 `--arch $(uname -m)` 重新打包 |
| 全员 token 突然失效 | AUTH_SECRET_KEY 变更 | 恢复备份的 .env；该密钥必须持久 |

## 8. 回滚

```bash
sudo bash /opt/tdsql-sqlcheck/current/../..../deploy/rollback.sh   # 或发布包内 deploy/rollback.sh
```

切回上一 `releases/` 目录并重启，**无需数据库降级**（表结构增量兼容，v1.0.2 无破坏性变更）。回滚后重跑 `verify_deploy.sh`。

## 9. 卸载（如需）

```bash
systemctl disable --now tdsql-sqlcheck
rm -f /etc/systemd/system/tdsql-sqlcheck.service && systemctl daemon-reload
rm -rf /opt/tdsql-sqlcheck          # 元数据库数据在 TDSQL 实例中，按需 DROP DATABASE
userdel sqlcheck
```
