# TDSQL数据库SQL审核工具 v1.1.0.0 测试用例规范 (Smoke / SIT / UAT)

本测试用例规范作为系统送测的质检验收依据，用于全面检验 TDSQL 数据库 SQL 审核工具在 `v1.1.0.0` 定版发布下的各项核心功能、稳定性防线与业务流闭环。

| **SM-02** | 静态资产 | 静态资源可访问性 | 服务已启动且在 8000 端口监听 | 依次在浏览器中访问：<br>1. `http://127.0.0.1:8000/`<br>2. `/static/js/app.js`<br>3. `/static/css/app.css` | 1. 成功返回首页 HTML 内容。<br>2. 成功返回 js/css 资源，响应状态码均为 200。 | Block |

## 三、 用户验收测试用例 (UAT Test Cases - 场景闭环)

UAT 测试从业务角色视角出发，执行完整的日常端到端工作流，验证业务闭环的顺畅程度。

### 场景一：系统管理员 (Admin) 权限分配与日常运维闭环
* **测试目的**：验证管理员进行用户角色配置、细化控制深度诊断权限的业务闭环。
* **业务流程**：
  1. **用户初始化**：使用 `admin` 登录，进入“系统管理”>“用户管理”，新建用户 `role_dba` (角色: dba)、`role_dev` (角色: developer)。
  2. **权限隔离微调**：进入“系统管理”>“权限矩阵”，为 `developer` 角色去掉“结构比对”、“应急诊断”和“运维工具箱”的可见权限勾选，点击保存。
  3. **登录验证**：使用 `role_dev` 登录，由于是首次登录，强制修改密码为 `DevPass123`，重新登录。进入系统后确认：
     - 主导航菜单中看不到“系统信息”、“权限矩阵”等管理模块。
     - 进入“深度诊断”页面，顶部标签页中完全没有“结构比对”、“应急诊断”、“运维工具箱”这三项。
  4. **全功能覆盖确认**：退出，使用 `role_dba` 首次登录并强制修改密码为 `DbaPass123`，重新登录。可以完整看到“深度诊断”下的所有 9 个子标签。

### 场景二：数据库管理员 (DBA) 实例纳管与每日巡检闭环
* **测试目的**：验证 DBA 从注册数据库、日常巡检采集、差异比对并导出分析报告的闭环。
* **业务流程**：
  1. **实例注册与测试**：使用 `role_dba` 登录系统，进入“实例与体检”>“实例管理”：
     - 点击“新增实例”，输入 TDSQL 集群的实例信息与对应的 monitordb 连接信息。
     - 点击“测试连接”，验证页面弹出“已连接”提示，且列表“状态”字段在 0.5 秒内同步更新为带绿点的“已连接”。
  2. **日常巡检数据采集**：进入“深度诊断”>“日常巡检与对比报告”：
     - 选择刚刚注册的实例，选择日期范围为 `7月18日 ~ 7月20日`（3天），点击“手动采集巡检数据”。
     - 确认采集成功，折线图成功渲染 3 天的性能趋势。
  3. **起止数据差异环比分析**：
     - 观察下方的差异比对表格，分析 7月18日 与 7月20日 两天实例与主机的 CPU峰值、内存使用率、慢 SQL 总数、磁盘空间的变化。
     - 修改“变化波动率”阈值滑块，确认表格中高亮黄色警告 `⚠️` 随着阈值调整而动态增减。
  4. **报告导出**：
     - 点击“导出HTML对比大屏”按钮，系统在新窗口打开集成了折线图、极简风格设计的交互式网页大屏报告，检查其排版与响应度。

### 场景三：业务开发人员 (Developer) SQL上线审核闭环
* **测试目的**：验证开发人员在线自主提交 SQL 语句、触发规则审核、查看上线检查清单的闭环。
* **业务流程**：
  1. **自主审核 SQL**：使用 `role_dev` 登录系统，进入“SQL审核”>“即时审核”：
     - 在文本框中输入几条测试 SQL（如带有未加索引的查询、含有 `SELECT *`、以及违反命名规范的 DDL）。
     - 点击“开始审核”，检查右侧即时返回的审核报告。
     - 确认列表中能够准确标出违规规则（如 `naming01`、`index02`），并给出修改建议。
  2. **多库模式检查**：
     - 该多库审核能力实为“上线检查”功能。进入“实例与体检”>“上线检查”：选择目标实例，不指定具体数据库过滤（留空），点击“开始检查”并“导出HTML报告”，确认可以一次性针对同一连接下的多个不同数据库执行审核。
  3. **查看上线清单与规则库**：
     - 进入“SQL审核”>“审核规则库”，查看当前生效的规则库，确认能够检索出目前支持的 119 条规则分类和详细判定标准。
