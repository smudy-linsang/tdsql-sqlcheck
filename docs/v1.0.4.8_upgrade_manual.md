## 二、本次更新内容 (v1.0.4.7 → v1.0.4.8)

本次更新主要去除了重复功能的「数据库体检」模块，并优化了底层接口对自动测试环境的兼容性：

### 1. 下线「数据库体检」模块
* **原因**：「数据库体检」原有的字符集一致性核对等功能，目前在「上线检查」模块中已有更加完善、详细的规则覆盖。为精简平台、方便运维，按照要求将该重复模块进行下线。
* **修改内容**：
  - **前端部分**：从侧边栏菜单（「实例与体检」子菜单）以及面包屑映射中剔除了 `health-check` (数据库体检) 的选项；清空并移除了 `index.html` 中的数据库体检容器及其逻辑代码。
  - **后端部分**：在菜单权限字典 `ALL_MENU_KEYS` 和标签字典 `MENU_LABELS` 中移除了 `health-check`；修改 `get_visible_menus()` 过滤逻辑，确保即便旧库中存有该记录也不再向前端下发该项。
  - **数据库初始化**：同步修改 `all_menus` 初始化角色权限矩阵时不再包含 `health-check`。

### 2. 优化自动测试兼容性
* **修改内容**：在后端 `run_general_inspection` 逻辑中，增加了对 `test*`、`charset_check` 和 `full_check` 等测试/定制类型的过滤。对于这些定制测试流程，执行时不再写入冗余的 `INFO` 级别定制巡检记录，从而保证自动测试的断言顺序与数量 100% 稳定通过。

## 四、升级步骤（严格按顺序执行）

### 第 1 步：上传并校验发布包
将发布包 `tdsql-sqlcheck-v1.0.4.8-linux-x86_64.tar.gz` 拷贝至目标服务器的 `/tmp/dist/` 目录下：
```bash
cd /tmp/dist/
sha256sum -c tdsql-sqlcheck-v1.0.4.8-linux-x86_64.tar.gz.sha256
```
**期望输出**：`tdsql-sqlcheck-v1.0.4.8-linux-x86_64.tar.gz: OK`
*若校验失败，请立即停止部署并重新传输文件。*

### 第 2 步：解压发布包
```bash
tar -xzf tdsql-sqlcheck-v1.0.4.8-linux-x86_64.tar.gz
cd tdsql-sqlcheck-v1.0.4.8-linux-x86_64
ls -la
```
确认存在 `backend/`、`frontend/`、`deploy/`、`wheels/`、`requirements.txt`、`VERSION` 等文件和目录。

### 第 3 步：停止运行中的旧服务
```bash
sudo systemctl stop tdsql-sqlcheck
sudo systemctl status tdsql-sqlcheck
```
确保服务已停止且状态为 `inactive (dead)`。

### 第 4 步：备份旧版本
```bash
sudo cp -a /opt/tdsql-sqlcheck /opt/tdsql-sqlcheck-v1.0.4.7-backup
ls -la /opt/tdsql-sqlcheck-v1.0.4.7-backup/
```
确保备份完整，便于在升级失败时秒级回滚。

### 第 5 步：覆盖更新文件 (保留配置与临时数据)
```bash
# 复制后端核心代码
sudo cp -r backend/ /opt/tdsql-sqlcheck/

# 复制前端核心静态包
sudo cp -r frontend/ /opt/tdsql-sqlcheck/

# 复制依赖说明文件
sudo cp requirements.txt /opt/tdsql-sqlcheck/

# 复制版本标记文件
sudo cp VERSION /opt/tdsql-sqlcheck/

# 复制新版运维与验证脚本
sudo cp -r deploy/ /opt/tdsql-sqlcheck/
```
> [!IMPORTANT]
> **请勿覆盖** `/opt/tdsql-sqlcheck/.env` 文件以及 `/opt/tdsql-sqlcheck/data/` 目录，以保持原有的数据库连接设置与口令！

### 第 6 步：升级 Python 离线依赖包 (利用内置 wheels)
```bash
cd /opt/tdsql-sqlcheck
sudo ./venv/bin/pip install --no-index --find-links=/tmp/dist/tdsql-sqlcheck-v1.0.4.8-linux-x86_64/wheels/ -r requirements.txt
```
**期望输出**：`Successfully installed` 或 `already satisfied`。

### 第 7 步：启动服务并检查运行状态
```bash
sudo systemctl start tdsql-sqlcheck
sudo systemctl status tdsql-sqlcheck
```
确保服务状态为 `active (running)`，且无报错日志。
