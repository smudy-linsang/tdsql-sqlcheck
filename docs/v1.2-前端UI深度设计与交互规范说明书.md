# TDSQL 数据库 SQL 审核平台 v1.2 前端 UI/UX 深度设计与交互规范说明书

> **设计目标**: 在保持现有金融级深色/浅色结合整体风格的大前提下，针对 v1.2 演进的各项功能进行像素级与全流程交互设计，确保全站无断点、无“死按钮”，提升易用性与操作反馈体验。  
> **设计基线**: Element Plus 设计语言 + TDSQL 深蓝导航主题 (`--nav-bg: #0f1e34`, `--brand-500: #2563eb`)  
> **文档版本**: v1.2-UI-UX-Final  
> **作者**: G 智能体 / 前端架构组  

---

## 一、 整体视觉语言与组件交互总则

### 1.1 品牌与色彩规范
全站延续银行级金融科技色彩体系：
* **导航与顶栏**: `--nav-bg` (`#0f1e34`) 搭配 `--nav-text` (`#b9c4d4`)，激活态采用高亮蓝线 `--nav-accent` (`#4c8dff`)。
* **品牌主色**: Primary 采用 `--brand-500` (`#2563eb`)，Hover 态为 `#1d4ed8`。
* **状态语义色**:
  - 成功/通过 (Success): `#16a34a` (浅绿 `#f0fdf4`)
  - 警告/中危 (Warning): `#d97706` (浅黄 `#fffbeb`)
  - 危险/高危 (Danger): `#dc2626` (浅红 `#fef2f2`)
  - 信息/提示 (Info): `#64748b` (浅灰 `#f8fafc`)

### 1.2 全站控件交互防断点原则 (Zero-Broken Principles)
1. **按钮防重复提交**: 所有的异步 API 调用按钮必须绑定 `:loading="loadingState[actionKey]"` 并在点击后处于 `disabled` 状态。
2. **全局空状态与加载态**:
   - 表格一律绑定 `v-loading="tableLoading"`。
   - 数据为空时展示 Element Plus 标准 `<el-empty description="暂无数据" />`。
3. **全局消息反馈 (Toast/Message)**:
   - 操作成功触发 `ElMessage.success({ message: "操作成功", duration: 2500 })`。
   - 操作失败触发 `ElMessage.error({ message: err.message || "请求失败", duration: 4000 })`。

## 二、 ToolBridge 运维工具箱界面 UI/UX 深度设计

### 2.1 整体布局
工作区分为三个区域：**分类筛选栏**、**工具卡片矩阵**、**执行抽屉/弹窗**。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│ 运维工具箱                                   [🔍 搜索工具...] [全部/磁盘/命令]│
├─────────────────────────────────────────────────────────────────────────────┤
│ ┌──────────────────────┐  ┌──────────────────────┐  ┌──────────────────────┐ │
│ │ 💾 磁盘性能测试 (fio)│  │ ⚡ sshpass 批量执行  │  │ 🔍 TDSQL 慢日志清理   │ │
│ │ 磁盘 IO/吞吐量压测   │  │ 多节点并行 Shell 指令│  │ 自动归档与日志压缩   │ │
│ │ [配置并运行] [历史] │  │ [配置并运行] [历史] │  │ [配置并运行] [历史] │ │
│ └──────────────────────┘  └──────────────────────┘  └──────────────────────┘ │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 工具卡片 UI 规格 (`el-card`)
- **Header**: 图标 (24px) + 工具名称 (16px bold) + 描述标签 (`el-tag` size="small" type="info")。
- **Body**: 工具简介说明 (13px, color: `--text-regular`, 限制 2 行省略号)。包含文件清单微缩提示。
- **Footer 操作栏**:
  - `[配置并运行]` 按钮: `type="primary"` `size="default"`，绑定 `@click="openToolDialog(tool)"`。
  - `[查看历史]` 按钮: `type="default"` `size="default"`，绑定 `@click="openHistoryDrawer(tool)"`。

### 2.3 在线参数配置对话框 (`el-dialog`)
- **标题**: `配置并运行: {tool.name}`
- **表单项 (`el-form`)**:
  1. `目标 TDSQL 实例`: `el-select`（下拉带搜索，展示实例名与 IP 端口），必填项。
  2. `并发线程数 / 参数`: `el-input-number` (min=1, max=32)。
  3. `自定义参数`: `el-input` placeholder="--duration 60"。
- **底部按钮**:
  - `[取消]`: `@click="toolDialogVisible = false"`。
  - `[提交运行]`: `type="primary"` `:loading="submittingTask"` `@click="submitToolTask"`。提交成功后自动关闭 Dialog 并推入实时日志抽屉。

### 2.4 实时日志流与控制台抽屉 (`el-drawer`)
- **抽屉属性**: `size="50%"` `title="工具执行控制台"` `direction="rtl"` `:before-close="handleDrawerClose"`。
- **顶部状态 Banner**:
  - 动态呈现：运行中 (`el-icon-loading` 蓝色)、成功 (`el-icon-success` 绿色)、失败 (`el-icon-error` 红色)。
  - 进度条: `el-progress` (`:percentage="taskProgress"` `:status="taskStatus"`).
- **控制台窗口 (`terminal-window`)**:
  - 采用黑底白字 terminal 风格 (`background: #0f172a`, `color: #38bdf8`, `font-family: monospace`)。
  - 支持 WebSocket / 轮询实时 append 输出日志，带自动滚动最底部 toggle (`Auto-scroll ON/OFF`)。
- **底部操作**:
  - `[终止任务]`: `type="danger"` `@click="killTask"` (任务运行中显示)。
  - `[下载诊断报告]`: `type="success"` `@click="downloadReport"` (任务完成后显示)。

## 三、 Monaco Editor 智能 SQL 审核及流式批处理 UI/UX 深度设计

### 3.1 Monaco Editor 编辑器卡片 (`MonacoEditor.vue`)
- **顶栏操作组 (`filter-bar`)**:
  - `[数据库类型]`: `el-select` (TDSQL MySQL / Oracle 兼容)。
  - `[目标实例]`: `el-select`。
  - `[即时审核]`: `type="primary"` `:loading="auditing"` `@click="doAudit"`。
  - `[一键格式化]`: `type="default"` `@click="formatSQL"`。
  - `[清空]`: `type="danger"` `text` `@click="clearSQL"`。
- **编辑器区域**:
  - 左侧边栏显示行号、语法折叠；
  - 违法项通过 `monaco.editor.setModelMarkers` 渲染下划波浪线（红色 Error、黄色 Warning）；
  - 鼠标悬停波浪线弹出带微型 Tag 的框：`[R001] WHERE 条件未包含主键` + 优化建议。

### 3.2 审核结果响应面板（右侧 35%）
- **顶部统计 Banner**:
  - 合格: 大号绿色 Icon + “审核通过 (0 违规)”。
  - 不合格: 大号红色 Icon + “检测到 3 项违规” (包含高危 `el-tag` 计数)。
- **违规规则折叠列表 (`el-collapse`)**:
  - 折叠项 Header: `[R002 High]` `第 5 行` `SELECT * 通配符限制`。
  - 折叠项 Body: 详细错误说明 + 推荐改写 SQL 片段（带一键复制 `[复制改写]` 按钮）。
  - **点击联动**: 点击任意违规项，Monaco Editor 自动调用 `editor.revealLineInCenter(line)` 并高亮该行。

### 3.3 流式大文件批处理模态框 (`el-dialog`)
- **触发入口**: 点击 SQL 审核页 `[批量文件流式审核]` 按钮。
- **上传交互**: `el-upload` 拖拽区域，支持 `.sql` / `.xml` 文件。
- **流式进度面板**:
  - 动态显示：`已解析语句数: 1,420 条 | 违规项: 12 条`。
  - 实时输出表格 (`el-table` max-height="300")，通过 NDJSON 流实时 `push` 新检测到的违规语句。
  - 底部操作: `[暂停]`、`[导出完整 PDF 报告]`。

## 四、 TDSQL 分布式架构检测可视化 UI/UX 深度设计

针对 TDSQL 专属分布式特性的前端交互呈现：

### 4.1 Shard Key 缺失可视化预警 Banner
在诊断结果顶部呈现黄色/红色警告框：
```html
<div class="dist-warning-banner danger">
  <el-icon><WarningFilled /></el-icon>
  <div class="banner-content">
    <div class="title">⚠️ 警告：当前 DML 语句缺少分片键 [order_id]</div>
    <div class="sub">该查询将被路由广播至全部 16 个 SET 节点，可能引发全表扫描与 CPU 飙升！</div>
  </div>
  <el-button type="warning" size="small" @click="showRouteDetail">查看路由推演</el-button>
</div>
```

### 4.2 广播表高危变更确认弹窗 (`el-message-box`)
在对广播表（Broadcast Table）触发 UPDATE / DELETE 时，界面不直接提交，而是弹出二次确认框：
```javascript
ElMessageBox.confirm(
  '当前目标表 [t_config] 为全局广播表，修改操作将对所有 SET 节点加分布式锁并同步更新。确定要在业务高峰期执行吗？',
  '高危广播表变更确认',
  { confirmButtonText: '确定提交', cancelButtonText: '取消', type: 'warning' }
)
```

## 五、 双 UI 模式无缝切换与体验流

### 5.1 切换入口与交互
在顶部 Header 用户下拉菜单内新增：
```html
<el-dropdown-item command="switch_ui_v2">✨ 切换到 Vue3 现代化版 (v2)</el-dropdown-item>
<el-dropdown-item command="switch_ui_v1">⚡ 切换到极简离线版 (v1)</el-dropdown-item>
```

### 5.2 切换逻辑
- 点击后将选中的 UI 版本保存至 `localStorage.setItem('preferred_ui', 'v2')`。
- 弹出加载提示 `ElLoading.service({ text: '正在加载 UI 界面...' })`。
- 动态跳转 URL 参数 `/?ui=v2` 或 `/?ui=v1`，无需重启后端即可平滑无缝切换。

## 六、 照图施工级前端组件与状态管理代码实现

### 6.1 ToolBridge 抽屉组件 `ToolConsoleDrawer.vue`
```vue
<template>
  <el-drawer
    v-model="visible"
    title="ToolBridge 运维控制台"
    size="50%"
    direction="rtl"
    :before-close="handleClose"
  >
    <div class="console-header">
      <div class="status-box">
        <el-tag :type="statusTagType" effect="dark" size="large">{{ taskStatus }}</el-tag>
        <span class="task-id">Run ID: {{ runId }}</span>
      </div>
      <el-progress :percentage="progress" :status="progressStatus" class="mt-12" />
    </div>

    <!-- 终端日志窗口 -->
    <div ref="terminalRef" class="terminal-window mt-16">
      <pre v-for="(line, idx) in logLines" :key="idx" class="log-line">{{ line }}</pre>
    </div>

    <template #footer>
      <div class="drawer-footer">
        <el-button v-if="taskStatus === 'RUNNING'" type="danger" :loading="killing" @click="killTask">
          终止任务
        </el-button>
        <el-button v-if="taskStatus === 'SUCCESS'" type="success" @click="downloadReport">
          下载诊断报告
        </el-button>
        <el-button @click="visible = false">关闭</el-button>
      </div>
    </template>
  </el-drawer>
</template>

<script setup>
import { ref, computed, nextTick } from 'vue';
import { ElMessage } from 'element-plus';

const visible = ref(false);
const runId = ref('');
const taskStatus = ref('RUNNING');
const progress = ref(30);
const logLines = ref([]);
const terminalRef = ref(null);
const killing = ref(false);

const statusTagType = computed(() => {
  if (taskStatus.value === 'RUNNING') return 'warning';
  if (taskStatus.value === 'SUCCESS') return 'success';
  return 'danger';
});

const progressStatus = computed(() => {
  if (taskStatus.value === 'SUCCESS') return 'success';
  if (taskStatus.value === 'FAILED') return 'exception';
  return '';
});

function appendLog(text) {
  logLines.value.push(text);
  nextTick(() => {
    if (terminalRef.value) {
      terminalRef.value.scrollTop = terminalRef.value.scrollHeight;
    }
  });
}

function killTask() {
  killing.value = true;
  setTimeout(() => {
    taskStatus.value = 'FAILED';
    appendLog('[SYSTEM] 任务已被用户强制终止。');
    killing.value = false;
    ElMessage.warning('任务已终止');
  }, 1000);
}

defineExpose({ visible, runId, taskStatus, progress, appendLog });
</script>

<style scoped>
.terminal-window {
  background: #0f172a;
  color: #38bdf8;
  font-family: var(--font-mono);
  padding: 16px;
  border-radius: 6px;
  height: 420px;
  overflow-y: auto;
}
.log-line {
  margin: 0;
  line-height: 1.5;
  white-space: pre-wrap;
}
.drawer-footer {
  display: flex;
  justify-content: flex-end;
  gap: 12px;
}
</style>
```
