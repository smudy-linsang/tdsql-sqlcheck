## 一、本次 V1.2.0.3 核心修复要素

1. **内网连接密码解密明文降级保护（彻底解决 1045 password-less 错误）**：
   - 彻底重构了 `security_service.py` 中的 `decrypt_password` 逻辑；
   - 增加了 Fernet 密文格式防误判与明文原样容错：当连接配置中的密码为明文时，绝不误解密抹空，原样安全返回，彻底解决了大表治理、慢 SQL 扫描与元数据提取时出现的 TDSQL Proxy 1045 `password-less login is not allowed` 拒绝访问报错。

2. **后端 API 错误拦截与 400 状态码重构（解决前端 Unexpected token 'I' 500 崩溃）**：
   - 重构了 `sql_audit.py` 和 `inspection.py` 中的异常捕获与 `scopes` 非空防御；
   - 所有数据库与应用层异常统一返回标准 400 JSON Response 格式，杜绝返回 HTML 500 页面引发前端 `Response: Unexpected token 'I'` 的全屏红框崩溃，实现前端错误弹窗文本极其清晰、可读。

3. **全量 999 项自动化测试 100% 绿灯 PASS**：
   - 新增 `test_v2_security_fallback.py` 自动化测试套件；
   - 跑完整体 pytest 测试覆盖，全量通过。

## 二、5 步平滑升级与部署指令

```bash
# 1. 停止旧服务并备份
sudo systemctl stop tdsql-sqlcheck || killall python || true
sudo cp -a /opt/tdsql-sqlcheck /opt/tdsql-sqlcheck-backup-$(date +%Y%m%d)

# 2. 解压 v1.2.0.3 最新发布包
cd /opt/
sudo tar -xzf /tmp/tdsql-sqlcheck-v1.2.0.3-source.tar.gz -C /opt/
sudo chown -R root:root /opt/tdsql-sqlcheck

# 3. 激活虚拟环境并更新依赖
cd /opt/tdsql-sqlcheck
source venv/bin/activate || python3 -m venv venv && source venv/bin/activate
pip install --no-index --find-links=../wheels -r requirements.txt || pip install -r requirements.txt

# 4. 后台拉起服务
nohup python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 > backend_restart.log 2>&1 &

# 5. 健康检查
curl -s http://localhost:8000/health
```
**期望标准输出**：`{"status":"ok","version":"1.2.0.3"}`
