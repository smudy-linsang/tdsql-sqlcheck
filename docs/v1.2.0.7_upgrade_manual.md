> **升级路径**: **v1.1.0.1 / v1.2.0.4 / v1.2.0.5 / v1.2.0.6 ➔ v1.2.0.7 (平滑无缝升级)**

## 一、本次 V1.2.0.7 核心修复与开发变更说明

### 1. 消除多角色登录/首次登录强制改口令时的“霸屏无权弹窗” (P0)
- 移除了网络底层拦截器 `apiFetch` 中针对 HTTP 403 响应强行弹出 Warning Toast 的代码。
- 页面数据请求重构为按需拉取，登录后首先获取 `visibleMenus`（账号可访问菜单），只拉取账号拥有权限的模块数据。
- 增加了首屏智能路由跳转：受限角色登录后，系统自动引导进入其拥有的第一个有效工作台。

### 2. 修复实例管理与体检下拉框数据加载逻辑 (P0)
- 修正了 `app.js` 中实例管理权限 Key (`instances`) 的匹配逻辑，恢复登录及页面切换时 `loadSavedConnections()` 的自动调用。实例列表正常在列表及下拉框中展示。

### 3. 内网离线交付与构建重构
- `requirements.txt` 中统一使用 `uvicorn>=0.34.0`，从源头解决 wheels 目录缺少 `uvloop` 阻断安装的问题。
- 预检和安装脚本自动扫描 `/opt/python311/python/bin` 便携 Python。
- 打包清单中补齐 `deploy/rollback.sh` 一键回滚脚本与包含全套 `1.2.0.7` 部署手册。

## 二、部署与升级操作步骤 (内网部署智能体适用)

### 1. 拷贝并校验升级包
将 `dist/tdsql-sqlcheck-v1.2.0.7-linux-x86_64.tar.gz` 及其 SHA256 校验文件拷贝至目标服务器：

```bash
cd /tmp
sha256sum -c tdsql-sqlcheck-v1.2.0.7-linux-x86_64.tar.gz.sha256
```

### 2. 解压 v1.2.0.7 最新发布包
```bash
sudo tar -xzf /tmp/tdsql-sqlcheck-v1.2.0.7-linux-x86_64.tar.gz -C /opt/
cd /opt/tdsql-sqlcheck-v1.2.0.7-linux-x86_64
```

### 3. 配置 .env 并安装
```bash
cp deploy/env.template deploy/.env
# 若为升级场景，请保留旧 .env 中的 AUTH_SECRET_KEY 以免用户掉线
sudo ./deploy/install.sh
```

## 四、回滚步骤（如遇异常）

若升级后需要退回上一稳定版本，直接运行根目录的 `rollback.sh` 即可：

```bash
sudo ./deploy/rollback.sh
```
