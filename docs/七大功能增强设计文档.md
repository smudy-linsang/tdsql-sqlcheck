## 一、功能1: Logo自定义上传

### 需求
系统界面左上角logo可由用户自定义上传图片替换。

### 设计
- **后端**: `POST/GET/DELETE /api/v1/admin/logo` 三个接口
  - 上传：接收 `UploadFile`，保存到 `frontend/static/img/custom-logo.png`（覆盖式，限2MB，支持png/jpg/svg/ico）
  - 获取：返回当前logo URL和是否自定义标志
  - 删除：删除自定义logo文件，恢复默认文字"T"
- **前端**: 顶栏品牌区 `<img v-if="logoUrl">` 替代 `<div>T</div>`；系统信息页加 `el-upload` 上传按钮 + "恢复默认"按钮

### 文件变更
- `backend/api/admin.py` — 新增3个路由
- `frontend/index.html` — 顶栏品牌区改为img + 系统信息页加上传区
- `frontend/static/js/app.js` — `logoUrl`状态 + `loadLogo`/`onLogoUpload`/`resetLogo`函数

## 二、功能2: 系统信息开关可配置

### 需求
系统信息页面的认证/脱敏/指标开关，用户能手动在前端切换。

### 设计
- **后端**: 新增 `system_config` 表（`config_key PK, config_value, updated_at`）
- **后端**: `GET/PUT /api/v1/admin/config` 接口读写开关
- **后端**: `config.py` 的 `auth_enabled()`/`data_masking_enabled()`/`metrics_enabled()` 改为优先读DB，回退环境变量
- **前端**: `el-descriptions` 中的开关值改为 `el-switch`，`@change` 调 `PUT /admin/config`

### 文件变更
- `backend/services/database.py` — 新建 `system_config` 表DDL
- `backend/config.py` — 新增 `_get_db_config()` + 三个开关函数改造
- `backend/api/admin.py` — 新增 `GET/PUT /config` 路由
- `frontend/index.html` — 系统信息页开关改为 `el-switch`
- `frontend/static/js/app.js` — `toggleSysConfig` 函数

## 六、功能6: 角色模块权限矩阵配置

### 需求
管理员能定义每个角色能看到哪些功能模块（含子模块），颗粒度覆盖所有模块和子模块。

### 设计
- **后端**: 新增 `role_permissions` 表（`role_id, menu_key, visible, PRIMARY KEY(role_id, menu_key)`）
- **后端**: `GET /api/v1/auth/role-permissions` 返回角色列表+菜单列表+权限矩阵
- **后端**: `PUT /api/v1/auth/role-permissions/{role_id}` 批量设置某角色可见菜单
- **后端**: `auth_service.py` 新增 `get_role_permissions`/`set_role_permissions`/`get_visible_menus` 函数
- **前端**: 新增"权限矩阵"页面，矩阵表格（行=角色，列=菜单项），每个单元格 `el-switch`

### 菜单key清单（22项，覆盖所有模块和子模块）
```
dashboard, audit-sql, file-audit, rules,
slow-tasks, slow-records, slow-schedule, explain,
instances, health-check, bigtable,
projects, rulesets, gate, monitor, inspection,
sys-users, sys-retention, sys-auditlog, sys-info,
sys-roles, sys-perms
```

### 文件变更
- `backend/services/database.py` — 新建 `role_permissions` 表DDL + 初始化数据
- `backend/services/auth_service.py` — 新增权限矩阵函数 + `ALL_MENU_KEYS`/`MENU_LABELS`常量
- `backend/api/auth.py` — 新增 `GET/PUT /role-permissions` 路由
- `frontend/index.html` — 新增权限矩阵页面
- `frontend/static/js/app.js` — `permsMatrixData`/`permsMenuList`/`loadPerms`/`onPermChange`

## 七、功能7: 角色CRUD管理

### 需求
角色不再限于预制4种，管理员可新增/删除自定义角色。

### 设计
- **后端**: 新增 `roles` 表（`role_id PK, role_name, is_builtin, description, created_at`）
- **后端**: 初始化4个内置角色记录
- **后端**: `GET /api/v1/auth/roles` 从DB动态读取
- **后端**: `POST /api/v1/auth/roles` 创建新角色（同时初始化权限矩阵为全不可见）
- **后端**: `DELETE /api/v1/auth/roles/{role_id}` 删除自定义角色（内置不可删，有用户引用不可删）
- **后端**: `auth_service.py` 的 `ROLES` 保持兼容，新增 `get_all_roles()`/`get_role_ids()` 从DB读取
- **前端**: 新增"角色管理"页面，表格 + 新建对话框 + 删除按钮

### 文件变更
- `backend/services/database.py` — 新建 `roles` 表DDL + 初始化4个内置角色
- `backend/services/auth_service.py` — `get_all_roles`/`create_custom_role`/`update_role`/`delete_role` 函数
- `backend/api/auth.py` — 角色CRUD路由 + `GET /visible-menus` 路由
- `frontend/index.html` — 新增角色管理页面 + 新建角色对话框
- `frontend/static/js/app.js` — `rolesList`/`roleDialog`/`loadRoles`/`createRole`/`deleteRole`
