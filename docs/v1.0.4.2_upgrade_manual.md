## 二、本次更新内容 (v1.0.4.1 → v1.0.4.2)

本次更新为热补丁调优版本，专门针对在大规模内网环境下扫描 MonitorDB 慢日志发生超时断连的 Bug 进行优化：

### 1. 独立调优 MonitorDB 的连接超时与读取超时
* **问题现象**：在内网部署后，新建扫描任务使用“全网慢SQL(monitordb)”数据源，由于监控库 `proxy_classes_analysis` 数据量大、聚合分析时间较长，导致频繁抛出报错：`monitordb: (2013, 'Lost connection to MySQL server during query (timed out)')`。
* **优化内容**：系统默认连接超时和读取超时（10秒）对集群级慢日志表扫描时间过短。本次更新在 [tdsql_connector.py](file:///c:/TDSQL_SQLCHECK/TDSQL-SQLCheck/backend/services/tdsql_connector.py) 中，针对 MonitorDB 短连接专门将**连接超时调大至至少 5 分钟 (300 秒)**，**查询读取超时调大至至少 15 分钟 (900 秒)**。
* **预期效果**：能够保障在大数据量或慢查询堆积的 TDSQL 集群上，慢 SQL 治理任务仍能稳定耗时聚合，不再抛出 PyMySQL 2013 连接断开异常。

## 四、升级步骤（严格按顺序执行）

### 第 1 步：上传并校验发布包
将发布包 `tdsql-sqlcheck-v1.0.4.2-linux-x86_64.tar.gz` 拷贝至目标服务器的 `/tmp/dist/` 目录下：
```bash
cd /tmp/dist/
sha256sum -c tdsql-sqlcheck-v1.0.4.2-linux-x86_64.tar.gz.sha256
```
**期望输出**：`tdsql-sqlcheck-v1.0.4.2-linux-x86_64.tar.gz: OK`
*若校验失败，请立即停止部署并重新传输文件。*

### 第 2 步：解压发布包
```bash
tar -xzf tdsql-sqlcheck-v1.0.4.2-linux-x86_64.tar.gz
cd tdsql-sqlcheck-v1.0.4.2-linux-x86_64
ls -la
```
确认存在 `backend/`、`frontend/`、`deploy/`、`wheels/`、`requirements.txt`、`VERSION` 等文件和目录。

### 第 3 步：停止运行中的 v1.0.4.1 服务
```bash
sudo systemctl stop tdsql-sqlcheck
sudo systemctl status tdsql-sqlcheck
```
确保服务已停止且状态为 `inactive (dead)`。

### 第 4 步：备份旧版本
```bash
sudo cp -a /opt/tdsql-sqlcheck /opt/tdsql-sqlcheck-v1.0.4.1-backup
ls -la /opt/tdsql-sqlcheck-v1.0.4.1-backup/
```
确保备份完整，便于在升级失败时秒级回滚。

### 第 5 步：覆盖更新文件 (保留配置与临时数据)
```bash
# 复制后端核心代码
sudo cp -r backend/ /opt/tdsql-sqlcheck/

# 复制前端核心静态包
sudo cp -r frontend/ /opt/tdsql-sqlcheck/

# 复制依赖说明文件
sudo cp requirements.txt /opt/tdsql-sqlcheck/

# 复制版本标记文件
sudo cp VERSION /opt/tdsql-sqlcheck/

# 复制新版运维与验证脚本
sudo cp -r deploy/ /opt/tdsql-sqlcheck/
```
> [!IMPORTANT]
> **请勿覆盖** `/opt/tdsql-sqlcheck/.env` 文件以及 `/opt/tdsql-sqlcheck/data/` 目录，以保持原有的数据库连接设置与口令！

### 第 6 步：升级 Python 离线依赖包 (利用内置 wheels)
```bash
cd /opt/tdsql-sqlcheck
sudo ./venv/bin/pip install --no-index --find-links=/tmp/dist/tdsql-sqlcheck-v1.0.4.2-linux-x86_64/wheels/ -r requirements.txt
```
**期望输出**：`Successfully installed` 或 `already satisfied`。

### 第 7 步：启动服务
```bash
sudo systemctl start tdsql-sqlcheck
sleep 5
sudo systemctl status tdsql-sqlcheck
```

### 第 8 步：执行部署验证脚本
```bash
cd /opt/tdsql-sqlcheck
sudo bash deploy/verify_deploy.sh
```
**期望输出**：
```text

  [PASS] 健康检查 /health

════ 验证结果: PASS=7 FAIL=0 ════
```
