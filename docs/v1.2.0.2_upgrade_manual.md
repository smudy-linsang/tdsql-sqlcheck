## 一、本次 V1.2.0.2 核心更新要素

1. **在线元数据 DDL 精准 Key 匹配提取修复**：
   - 彻底废除了依赖字典隐式索引顺序的风险逻辑，重构为按 `Create Table` / `Create View` 显式精确 Key 匹配及非空保护；
   - 解决内网复杂 DDL / VIEW 提取被提前截断的源头缺陷。

2. **多段 SQL 智能切割防粘连防护**：
   - 在 `_split_sql_file` 中新增基于 `-- SQL Object:` / `-- Table:` / `-- View:` 标头注释的防粘连安全分隔机制；
   - 即使在线文件遭遇格式异常或丢失分号，也能实现对象间精准独立切分，杜绝多表 DDL 连带粘连。

3. **残缺/语法错误 SQL 阻断与 E999_SYNTAX_ERROR 告警**：
   - 在核心审核引擎 `RuleChecker` 中建立严密防线：任何语法分析失败或中途截断的不完整 SQL 严禁标记为通过；
   - 自动注入 `ERROR` 级别的 `E999_SYNTAX_ERROR` 严重违规阻断通过，并在审核报告中明确标红提醒。

4. **100% 自动化测试回归与打包规范**：
   - 全量 997 项自动化测试（新增 `test_v2_syntax_truncation.py`）一次性全部绿灯通过；
   - 包含全量 Python 离线 Wheels 包的标准 20MB+ 离线发布包。

## 二、5 步平滑升级与部署指令

```bash
# 1. 停止旧服务与备份
sudo systemctl stop tdsql-sqlcheck || killall python || true
sudo cp -a /opt/tdsql-sqlcheck /opt/tdsql-sqlcheck-backup-$(date +%Y%m%d)

# 2. 解压 v1.2.0.2 最新发布包
cd /opt/
sudo tar -xzf /tmp/tdsql-sqlcheck-v1.2.0.2-source.tar.gz -C /opt/
sudo chown -R root:root /opt/tdsql-sqlcheck

# 3. 激活虚拟环境并更新依赖
cd /opt/tdsql-sqlcheck
source venv/bin/activate || python3 -m venv venv && source venv/bin/activate
pip install --no-index --find-links=../wheels -r requirements.txt || pip install -r requirements.txt

# 4. 后台拉起服务
nohup python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 > backend_restart.log 2>&1 &

# 5. 健康检查
curl -s http://localhost:8000/health
```
**期望标准输出**：`{"status":"ok","version":"1.2.0.2"}`
