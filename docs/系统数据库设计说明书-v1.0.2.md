## 一、数据库概述

### 1.1 数据库选型

本项目使用 **SQLite** 作为本地持久化存储，理由如下：
- 零运维：无需独立数据库进程，单文件部署
- 并发友好：WAL模式支持并发读 + 串行写，满足审核工具的读写模式
- 便携性：数据库文件随Docker卷挂载，易于备份与迁移
- 性能足够：审核历史、慢SQL记录等场景写入量可控（日均千级）

### 1.2 数据库文件

| 文件 | 路径 | 说明 |
|-----|------|------|
| 主数据库 | `data/tdsql_check.db` | 所有业务数据 |
| 备份目录 | `data/backup/` | 每日自动备份 |
| 报告目录 | `data/reports/` | PDF/HTML报告文件 |

### 1.3 连接配置

```python
# SQLite连接参数
PRAGMA journal_mode = WAL;        # WAL模式，提升并发读写
PRAGMA busy_timeout = 5000;       # 写冲突时等待5秒
PRAGMA foreign_keys = ON;         # 启用外键约束
PRAGMA synchronous = NORMAL;      # WAL模式下NORMAL即可
```

## 二、表结构详细设计

### T01. slow_queries — 慢SQL记录

**说明：** 存储从TDSQL抓取或手动录入的慢SQL记录及其分析结果。

```sql
CREATE TABLE slow_queries (
    id                  INTEGER PRIMARY KEY AUTOINCREMENT,
    fingerprint         TEXT NOT NULL,              -- SQL指纹
    sql_text            TEXT NOT NULL,              -- 原始SQL文本
    normalized_sql      TEXT DEFAULT '',            -- 格式化SQL
    db_name             TEXT DEFAULT '',            -- 数据库名
    connection_id       TEXT DEFAULT '',            -- TDSQL连接ID
    project_id          TEXT DEFAULT '',            -- 项目ID
    
    -- 执行统计
    exec_count          INTEGER DEFAULT 0,          -- 执行次数
    total_time_ms       REAL DEFAULT 0,             -- 总耗时(ms)
    avg_time_ms         REAL DEFAULT 0,             -- 平均耗时(ms)
    max_time_ms         REAL DEFAULT 0,             -- 最大耗时(ms)
    rows_examined       INTEGER DEFAULT 0,          -- 扫描行数
    rows_sent           INTEGER DEFAULT 0,          -- 返回行数
    lock_time_ms        REAL DEFAULT 0,             -- 锁等待时间(ms)
    
    -- 时间信息
    first_seen          TEXT,                       -- 首次出现时间
    last_seen           TEXT,                       -- 最近出现时间
    
    -- 分析结果
    problem_type        TEXT DEFAULT '',            -- 问题类型(逗号分隔)
    severity            TEXT DEFAULT 'INFO',        -- 严重级别: INFO/WARNING/ERROR
    root_cause          TEXT DEFAULT '',            -- 根因分析
    suggestion          TEXT DEFAULT '',            -- 优化建议
    optimized_sql       TEXT DEFAULT '',            -- 优化后SQL
    distributed_analysis TEXT DEFAULT '',            -- 分布式分析结果JSON
    index_suggestions   TEXT DEFAULT '[]',          -- 索引建议JSON
    rewrite_suggestions TEXT DEFAULT '[]',          -- 改写建议JSON
    
    -- 状态管理
    status              TEXT DEFAULT 'pending',     -- pending/analyzing/analyzed/optimizing/optimized/closed
    assigned_to         TEXT DEFAULT '',            -- 处理人
    
    -- 元数据
    analysis_json       TEXT DEFAULT '{}',          -- 完整分析结果JSON
    created_at          TEXT DEFAULT (datetime('now')),
    updated_at          TEXT DEFAULT (datetime('now'))
);

CREATE INDEX idx_slow_fingerprint ON slow_queries(fingerprint);
CREATE INDEX idx_slow_db ON slow_queries(db_name);
CREATE INDEX idx_slow_status ON slow_queries(status);
CREATE INDEX idx_slow_connection ON slow_queries(connection_id);
CREATE INDEX idx_slow_project ON slow_queries(project_id);
CREATE INDEX idx_slow_last_seen ON slow_queries(last_seen);
CREATE INDEX idx_slow_severity ON slow_queries(severity);
```

**与V0.4差异：** 新增 `connection_id`、`project_id`、`normalized_sql`、`distributed_analysis`、`index_suggestions`、`rewrite_suggestions`、`assigned_to` 字段。

### T02. audit_history — 审核历史

**说明：** 记录每次审核操作的汇总信息。

```sql
CREATE TABLE audit_history (
    id                  INTEGER PRIMARY KEY AUTOINCREMENT,
    audit_type          TEXT NOT NULL,              -- sql/file/upload/webhook/cli
    source              TEXT DEFAULT '',            -- 来源(文件名/MR标题等)
    project_id          TEXT DEFAULT '',            -- 项目ID
    connection_id       TEXT DEFAULT '',            -- TDSQL连接ID(元数据增强时)
    
    -- 审核汇总
    total_sql           INTEGER DEFAULT 0,          -- SQL总数
    passed              INTEGER DEFAULT 0,          -- 通过数
    failed              INTEGER DEFAULT 0,          -- 未通过数
    error_count         INTEGER DEFAULT 0,          -- ERROR级别数
    warning_count       INTEGER DEFAULT 0,          -- WARNING级别数
    pass_rate           REAL DEFAULT 0,             -- 通过率
    
    -- 门禁结果
    gate_passed         INTEGER DEFAULT NULL,       -- 门禁是否通过(0/1/NULL=未评估)
    gate_detail         TEXT DEFAULT '',            -- 门禁详情
    
    -- 审核结果摘要
    top_violations      TEXT DEFAULT '[]',          -- TopN违规规则JSON
    results_summary     TEXT DEFAULT '{}',          -- 结果摘要JSON
    
    created_at          TEXT DEFAULT (datetime('now'))
);

CREATE INDEX idx_audit_type ON audit_history(audit_type);
CREATE INDEX idx_audit_project ON audit_history(project_id);
CREATE INDEX idx_audit_created ON audit_history(created_at);
CREATE INDEX idx_audit_gate ON audit_history(gate_passed);
```

**与V0.4差异：** 新增 `project_id`、`connection_id`、`gate_passed`、`gate_detail`、`top_violations`、`results_summary` 字段。原 `results_json` 字段拆分到T03表。

### T03. audit_results — 审核结果详情

**说明：** 存储每次审核中每条SQL的详细违规信息，支持细粒度查询与趋势分析。

```sql
CREATE TABLE audit_results (
    id                  INTEGER PRIMARY KEY AUTOINCREMENT,
    audit_history_id    INTEGER NOT NULL,           -- 关联audit_history.id
    sql_text            TEXT NOT NULL,              -- SQL文本
    sql_type            TEXT DEFAULT '',            -- SQL类型
    line_number         INTEGER,                    -- 行号
    file_path           TEXT DEFAULT '',            -- 文件路径
    passed              INTEGER DEFAULT 1,          -- 是否通过(0/1)
    
    -- 违规详情(一条SQL可能多条违规，存为JSON)
    violations_json     TEXT DEFAULT '[]',          -- [{rule_id, category, severity, message, suggestion}]
    error_count         INTEGER DEFAULT 0,
    warning_count       INTEGER DEFAULT 0,
    
    -- 触发的规则ID列表(便于规则排名统计)
    triggered_rules     TEXT DEFAULT '',            -- 逗号分隔的规则ID
    
    created_at          TEXT DEFAULT (datetime('now')),
    
    FOREIGN KEY (audit_history_id) REFERENCES audit_history(id) ON DELETE CASCADE
);

CREATE INDEX idx_results_history ON audit_results(audit_history_id);
CREATE INDEX idx_results_passed ON audit_results(passed);
CREATE INDEX idx_results_rules ON audit_results(triggered_rules);
```

### 4.2 迁移执行策略

```python
def migrate_v04_to_v10(db_path: str):
    """执行V0.4到V1.0的数据库迁移"""
    conn = sqlite3.connect(db_path)
    
    # 检查当前版本
    version = conn.execute(
        "SELECT value FROM schema_version WHERE key='version'"
    ).fetchone()
    
    if version and version[0] >= "1.0":
        return  # 已迁移
    
    # 备份数据库
    backup_path = db_path + f".v04_backup_{datetime.now().strftime('%Y%m%d%H%M%S')}"
    shutil.copy2(db_path, backup_path)
    
    # 执行迁移SQL
    migration_sql = load_migration_script("migration_v04_to_v10.sql")
    conn.executescript(migration_sql)
    
    # 更新版本号
    conn.execute(
        "INSERT OR REPLACE INTO schema_version(key, value) VALUES('version', '1.0')"
    )
    conn.commit()
    conn.close()
```
