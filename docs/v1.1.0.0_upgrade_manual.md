## 二、本次更新内容 (v1.0.5.0 → v1.1.0.0)

本次更新为本阶段的定版发布，汇总合并了近期所有的业务优化与稳定性增强补丁：

### 1. 废弃菜单项彻底清理与下线
* **说明**：系统彻底下线了“数据库体检”（`health-check`）模块。
* **效果**：数据库启动初始化时，会自动清除 `role_permissions` 中所有包含 `health-check` 的过期权限数据，保证权限矩阵的干净整洁。

### 2. 深度诊断子模块权限颗粒化细分
* **说明**：将「深度诊断」下的 9 个子模块（集群巡检、日常巡检与对比报告、索引体检、结构比对、应急诊断、SQL分析、网关日志分析、PDF报告与大屏、运维工具箱）作为独立的权限项引入“权限矩阵”。
* **效果**：支持管理员在「权限矩阵」里自由为不同角色单独配置这 9 个子功能的可见性与访问权限，且支持存量角色权限的自动补齐与平滑过渡。

### 3. 日常巡检单次手动采集范围限制 (3天)
* **说明**：限制单次手动采集的时间跨度最长为 3 天，以防止内网超大规模集群或监控库数据量过大时，用户因选择过长的采集日期范围导致后台拉取任务堆积、独占服务引起卡死。

### 4. SQL分析模块平均耗时单位修正 (s → ms)
* **说明**：将“深度诊断”>“SQL分析”中的表格“平均耗时(s)”修正为“平均耗时(ms)”，通过在前端将秒换算为毫秒展示（保留两位小数），极大提升了可读性，并与系统其它监控模块保持一致。

## 三、数据库结构变更

本次升级包含**元数据库结构自动增量订正**。
在服务启动时，后端会自动执行以下 SQL 订正，无需运维人员手动执行：
1. 清除废弃菜单项：
   `DELETE FROM role_permissions WHERE menu_key NOT IN (...)`
2. 自动检查已有角色并补齐 9 个深度诊断子菜单的默认权限项：
   `INSERT INTO role_permissions(role_id, menu_key, visible) VALUES (...)`

## 四、升级步骤（严格按顺序执行）

### 第 1 步：上传并校验发布包
将发布包 `tdsql-sqlcheck-v1.1.0.0-linux-x86_64.tar.gz` 拷贝至目标服务器的 `/tmp/dist/` 目录下：
```bash
cd /tmp/dist/
sha256sum -c tdsql-sqlcheck-v1.1.0.0-linux-x86_64.tar.gz.sha256
```
**期望输出**：`tdsql-sqlcheck-v1.1.0.0-linux-x86_64.tar.gz: OK`
*若校验失败，请立即停止部署并重新传输文件。*

### 第 2 步：解压发布包
```bash
tar -xzf tdsql-sqlcheck-v1.1.0.0-linux-x86_64.tar.gz
cd tdsql-sqlcheck-v1.1.0.0-linux-x86_64
ls -la
```
确认存在 `backend/`、`frontend/`、`deploy/`、`wheels/`、`requirements.txt`、`VERSION` 等文件和目录。

### 第 3 步：停止运行中的旧服务
```bash
sudo systemctl stop tdsql-sqlcheck
sudo systemctl status tdsql-sqlcheck
```
确保服务已停止且状态为 `inactive (dead)`。

### 第 4 步：备份旧版本
```bash
sudo cp -a /opt/tdsql-sqlcheck /opt/tdsql-sqlcheck-v1.0.5.0-backup
ls -la /opt/tdsql-sqlcheck-v1.0.5.0-backup/
```
确保备份完整，便于在升级失败时秒级回滚。

### 第 5 步：覆盖更新文件 (保留配置与临时数据)
```bash
# 复制后端核心代码
sudo cp -r backend/ /opt/tdsql-sqlcheck/

# 复制前端核心静态包
sudo cp -r frontend/ /opt/tdsql-sqlcheck/

# 复制依赖说明文件
sudo cp requirements.txt /opt/tdsql-sqlcheck/

# 复制版本标记文件
sudo cp VERSION /opt/tdsql-sqlcheck/

# 复制新版运维与验证脚本
sudo cp -r deploy/ /opt/tdsql-sqlcheck/
```
> [!IMPORTANT]
> **请勿覆盖** `/opt/tdsql-sqlcheck/.env` 文件以及 `/opt/tdsql-sqlcheck/data/` 目录，以保持原有的数据库连接设置与口令！

### 第 6 步：升级 Python 离线依赖包 (利用内置 wheels)
```bash
cd /opt/tdsql-sqlcheck
sudo ./venv/bin/pip install --no-index --find-links=/tmp/dist/tdsql-sqlcheck-v1.1.0.0-linux-x86_64/wheels/ -r requirements.txt
```
**期望输出**：`Successfully installed` 或 `already satisfied`。

### 第 7 步：启动服务并检查状态
```bash
sudo systemctl start tdsql-sqlcheck
sudo systemctl status tdsql-sqlcheck
```
确保服务状态为 `active (running)` 且无错误日志。
