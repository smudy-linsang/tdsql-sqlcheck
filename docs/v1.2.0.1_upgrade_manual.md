## 二、本次 V1.2.0.1 定版核心更新要素

1. **Python 全版本运行兼容性修复（阻断缺陷 BUG-V12-01）**：
   - 补全 `backend/main.py` 中的 `from typing import Optional` 导入，彻底解决在 Python 3.9/3.11 及银河麒麟默认 Python 环境下的服务启动崩溃问题。
2. **在线元数据审核记录重复优化（中危缺陷 BUG-V12-02）**：
   - 优化 `audit_service.audit_file_content` 审计历史落库机制，实现在线元数据审核单条精准落库，消除历史列表重复污染。
3. **RBAC 鉴权安全隔离强化（BUG-RBAC-01 & ISSUE-T01）**：
   - 在 API 路由层与数据库菜单层构建**双层防御体系**，禁止 DBA 角色与 Developer 越权访问操作/审计日志 (`/api/v1/admin/operation-logs`)；
   - 自动化测试集补充 DBA 角色权限限制用例。
4. **工具箱运维调度严谨化（ISSUE-V12-03）**：
   - 修复 NameError 隐患，明确标注离线运维任务“已成功登记调度（离线执行桩）”，并自动填充标准 Python `datetime` 时间与操作员账号。
5. **项目结构精简与规范整理（ISSUE-V12-05）**：
   - 规范清理了连接器模块中未引用的无用代码文件，导出结构精简清晰。
6. **全量 995 项自动化测试 100% 绿灯 PASS**：
   - 单元测试、规则引擎、SIT/UAT 验收全量测试一次性通过。

## 三、傻瓜式 5 步升降级与部署指令

### 第 1 步：备份旧版本文件与数据库

```bash
# 停止正在运行的服务
sudo systemctl stop tdsql-sqlcheck || killall python

# 备份程序文件
sudo cp -a /opt/tdsql-sqlcheck /opt/tdsql-sqlcheck-v1.1-backup-$(date +%Y%m%d)

# 备份旧版数据库 (如果使用 SQLite)
sudo cp /opt/tdsql-sqlcheck/data/tdsql_sqlcheck.db /opt/tdsql-sqlcheck-v1.1-backup-$(date +%Y%m%d)/
```

### 第 2 步：解压最新发布包

```bash
cd /opt/
sudo tar -xzf /tmp/tdsql-sqlcheck-v1.2.0.1-source.tar.gz -C /opt/
sudo chown -R root:root /opt/tdsql-sqlcheck
```

### 第 3 步：离线 Python 依赖检查与虚拟环境安装

```bash
cd /opt/tdsql-sqlcheck

# 如果已有虚拟环境直接激活，无网环境无需重复下载包
source venv/bin/activate || python3 -m venv venv && source venv/bin/activate

# 离线批量安装/更新 wheels 包 (内网环境)
pip install --no-index --find-links=../wheels -r requirements.txt
```

### 第 4 步：服务启动与健康检查

```bash
# 后台拉起 uvicorn 服务
nohup python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 > backend_restart.log 2>&1 &

# 检查服务探针健康度
curl -s http://localhost:8000/health
```
**标准健康响应**：`{"status":"ok","version":"1.2.0.1"}`

### 第 5 步：快速验证

1. 浏览器打开 `http://<服务器IP>:8000`；
2. 登录 `admin` (初始密码自动生成或为环境变量指定)；
3. 点击导航栏 **「SQL 审核」 -> 「在线元数据审核」**，验证元数据反向拉取、审查及 HTML 报告生成与下载功能。
