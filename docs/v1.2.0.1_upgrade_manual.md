## 二、从 v1.1.0.1 ➔ v1.2.0.1 全量变更要素清单

从内网现运行的 **v1.1.0.1** 跨版本升级至 **v1.2.0.1**，系统包含了 **V1.2 大版本演进的全量能力** 以及 **质检整改定版的全量修复**：

### 1. 核心大版本演进特性 (v1.2.0.0 累积)
- **数据库 Schema 增量 Migration 引擎 (C1)**：自动版本号校验与 checksum 校验，零人工干预完成数据库演进；
- **TDSQL 深度分布式规则引擎 (E1)**：专门针对 TDSQL 架构检查 Shard Key 缺失广播扫描、GSI 延迟风险与大表隐患；
- **全新「在线元数据审核」子模块 (F1)**：支持反向拉取 Live 数据库 `TABLES/VIEWS` DDL，规则审核与 HTML/SQL 报告持久化导出下载；
- **架构拆分与性能提升 (C2 + C3)**：解耦 Parser 与 Connector 模块，提升规则分析效率。

### 2. 质检验收定版全量修复与增强 (v1.2.0.1 补丁)
- **Python 3.9/3.11 全版本兼容 (BUG-V12-01)**：解决银河麒麟 Linux 生产环境默认 Python 启动崩溃问题；
- **在线元数据历史重复去重 (BUG-V12-02)**：优化审计历史落盘，消除重复双写；
- **RBAC 双层安全隔离防线 (BUG-RBAC-01)**：API 路由与菜单树双重阻断 DBA 越权访问操作日志；
- **运维工具箱安全与错误修复 (ISSUE-V12-03)**：修复变量 NameError 隐患，明确标注“离线执行桩”，跨库 SQLite/MySQL 绝对安全兼容；
- **代码结构规范化与死代码清理 (ISSUE-V12-05)**：移除无用连接器文件，结构更加干净高效。

## 三、内网升级保姆级 5 步全指令 (跨版本 v1.1.0.1 ➔ v1.2.0.1)

### 第 1 步：停止正在运行的 v1.1.0.1 旧服务并备份

```bash
# 停止旧服务
sudo systemctl stop tdsql-sqlcheck || killall python || pkill -f uvicorn

# 备份 v1.1.0.1 应用文件与数据
sudo cp -a /opt/tdsql-sqlcheck /opt/tdsql-sqlcheck-v1.1.0.1-backup-$(date +%Y%m%d)
```

### 第 2 步：上传并解压 v1.2.0.1 离线安装包

```bash
cd /opt/
sudo tar -xzf /tmp/tdsql-sqlcheck-v1.2.0.1-source.tar.gz -C /opt/
sudo chown -R root:root /opt/tdsql-sqlcheck
```

### 第 3 步：进入运行目录并激活 Python 环境

```bash
cd /opt/tdsql-sqlcheck

# 激活现有的 venv 虚拟环境
source venv/bin/activate || python3 -m venv venv && source venv/bin/activate

# 批量更新离线依赖包
pip install --no-index --find-links=../wheels -r requirements.txt || pip install -r requirements.txt
```

### 第 4 步：后台启动并触发自动 DDL 迁移

```bash
# 后台拉起最新服务
nohup python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 > backend_restart.log 2>&1 &

# 稍等 2 秒等待数据库自动 Migration 完成
sleep 2
```

### 第 5 步：探针验证升级结果

```bash
curl -s http://localhost:8000/health
```
**预期标准响应**：`{"status":"ok","version":"1.2.0.1"}`

验证成功！系统已顺利从 **v1.1.0.1** 跨版本升级至 **v1.2.0.1**！
