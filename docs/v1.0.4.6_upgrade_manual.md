## 二、本次更新内容 (v1.0.4.5 → v1.0.4.6)

本次更新针对慢 SQL 记录管理页面的筛选项进行了修正：

### 1. 修正慢 SQL 记录页面“级别”筛选项
* **问题现象**：在前端的 **「慢SQL治理」 > 「慢SQL记录」** 页面中，右上角提供了一个“级别”筛选项，下拉菜单包含三个可选项：`CRITICAL`、`WARNING`、`INFO`。然而，我们的慢 SQL 规则和扫描体系中只定义了 `ERROR`、`WARNING`、`INFO`，并没有使用 `CRITICAL` 级别，导致在需要检索高危慢查询时无法正确过滤。
* **修改内容**：
  - 更新前端模板 [index.html](file:///c:/TDSQL_SQLCHECK/TDSQL-SQLCheck/frontend/index.html) 的筛选项配置：将原本错误的选项 `<el-option label="CRITICAL" value="CRITICAL"></el-option>` 替换为了符合我方规则体系的正确等级选项 **`<el-option label="ERROR" value="ERROR"></el-option>`**。
  - 用户现在可以正常通过选择 `ERROR` 级别，精确筛选出最严重的慢 SQL。

## 四、升级步骤（严格按顺序执行）

### 第 1 步：上传并校验发布包
将发布包 `tdsql-sqlcheck-v1.0.4.6-linux-x86_64.tar.gz` 拷贝至目标服务器的 `/tmp/dist/` 目录下：
```bash
cd /tmp/dist/
sha256sum -c tdsql-sqlcheck-v1.0.4.6-linux-x86_64.tar.gz.sha256
```
**期望输出**：`tdsql-sqlcheck-v1.0.4.6-linux-x86_64.tar.gz: OK`
*若校验失败，请立即停止部署并重新传输文件。*

### 第 2 步：解压发布包
```bash
tar -xzf tdsql-sqlcheck-v1.0.4.6-linux-x86_64.tar.gz
cd tdsql-sqlcheck-v1.0.4.6-linux-x86_64
ls -la
```
确认存在 `backend/`、`frontend/`、`deploy/`、`wheels/`、`requirements.txt`、`VERSION` 等文件和目录。

### 第 3 步：停止运行中的旧服务
```bash
sudo systemctl stop tdsql-sqlcheck
sudo systemctl status tdsql-sqlcheck
```
确保服务已停止且状态为 `inactive (dead)`。

### 第 4 步：备份旧版本
```bash
sudo cp -a /opt/tdsql-sqlcheck /opt/tdsql-sqlcheck-v1.0.4.5-backup
ls -la /opt/tdsql-sqlcheck-v1.0.4.5-backup/
```
确保备份完整，便于在升级失败时秒级回滚。

### 第 5 步：覆盖更新文件 (保留配置与临时数据)
```bash
# 复制后端核心代码
sudo cp -r backend/ /opt/tdsql-sqlcheck/

# 复制前端核心静态包
sudo cp -r frontend/ /opt/tdsql-sqlcheck/

# 复制依赖说明文件
sudo cp requirements.txt /opt/tdsql-sqlcheck/

# 复制版本标记文件
sudo cp VERSION /opt/tdsql-sqlcheck/

# 复制新版运维与验证脚本
sudo cp -r deploy/ /opt/tdsql-sqlcheck/
```
> [!IMPORTANT]
> **请勿覆盖** `/opt/tdsql-sqlcheck/.env` 文件以及 `/opt/tdsql-sqlcheck/data/` 目录，以保持原有的数据库连接设置与口令！

### 第 6 步：升级 Python 离线依赖包 (利用内置 wheels)
```bash
cd /opt/tdsql-sqlcheck
sudo ./venv/bin/pip install --no-index --find-links=/tmp/dist/tdsql-sqlcheck-v1.0.4.6-linux-x86_64/wheels/ -r requirements.txt
```
**期望输出**：`Successfully installed` 或 `already satisfied`。

### 第 7 步：启动服务
```bash
sudo systemctl start tdsql-sqlcheck
sleep 5
sudo systemctl status tdsql-sqlcheck
```

### 第 8 步：执行部署验证脚本
```bash
cd /opt/tdsql-sqlcheck
sudo bash deploy/verify_deploy.sh
```
**期望输出**：
```text

  [PASS] 健康检查 /health

  ...
════ 验证结果: PASS=7 FAIL=0 ════
部署验证全部通过 ✔
```
