## 二、本次更新内容 (v1.0.4.8 → v1.0.4.9)

本次更新主要优化了「深度诊断」中「日常巡检与对比报告」及「集群巡检」由于 monitordb 历史监控表数据量过大导致数据采集耗时极长（卡死、转圈数小时）的问题：

### 1. 深度巡检指标采集性能优化 (解决卡死、慢问题)
* **问题现象**：在内网大规模集群或 monitordb 历史库数据量非常大的场景下，点击“手动采集巡检数据”或发起集群巡检时，系统因需要对海量历史数据分区表（如 `m_data_YYYYMMDD`）执行多次多表 `JOIN` 和带有 `LIKE '%node%'` 的全表扫描模糊匹配，导致单个指标查询非常慢。多次顺序循环查询累加，造成前端长期处于“转圈”挂起状态，无法获取采集结果。
* **修改与优化内容**：
  - **Pre-cache 预解析映射**: 在执行每日巡检采集时，首先通过单次快速查询一次性拉取 `m_data_cur` 中所有指标对象的 `f_id` 映射，建立 `(mid, key) -> f_id` 内存缓存，避免网络 roundtrips 与复杂的 `JOIN` 计算。
  - **基于主键/索引的精细查询**: 彻底废除原本 `m_data_YYYYMMDD` 与 `m_data_cur` 的模糊 `JOIN` 关联查询。对于缓存命中 `f_id` 的指标，直接使用 `f_id = %s` 执行单表主键定位。
  - **效果**: 采集接口执行速度从**数小时**骤降至**毫秒级/秒级**，完美规避了大库聚合卡死和超时风险。

## 四、升级步骤（严格按顺序执行）

### 第 1 步：上传并校验发布包
将发布包 `tdsql-sqlcheck-v1.0.4.9-linux-x86_64.tar.gz` 拷贝至目标服务器的 `/tmp/dist/` 目录下：
```bash
cd /tmp/dist/
sha256sum -c tdsql-sqlcheck-v1.0.4.9-linux-x86_64.tar.gz.sha256
```
**期望输出**：`tdsql-sqlcheck-v1.0.4.9-linux-x86_64.tar.gz: OK`
*若校验失败，请立即停止部署并重新传输文件。*

### 第 2 步：解压发布包
```bash
tar -xzf tdsql-sqlcheck-v1.0.4.9-linux-x86_64.tar.gz
cd tdsql-sqlcheck-v1.0.4.9-linux-x86_64
ls -la
```
确认存在 `backend/`、`frontend/`、`deploy/`、`wheels/`、`requirements.txt`、`VERSION` 等文件和目录。

### 第 3 步：停止运行中的旧服务
```bash
sudo systemctl stop tdsql-sqlcheck
sudo systemctl status tdsql-sqlcheck
```
确保服务已停止且状态为 `inactive (dead)`。

### 第 4 步：备份旧版本
```bash
sudo cp -a /opt/tdsql-sqlcheck /opt/tdsql-sqlcheck-v1.0.4.8-backup
ls -la /opt/tdsql-sqlcheck-v1.0.4.8-backup/
```
确保备份完整，便于在升级失败时秒级回滚。

### 第 5 步：覆盖更新文件 (保留配置与临时数据)
```bash
# 复制后端核心代码
sudo cp -r backend/ /opt/tdsql-sqlcheck/

# 复制前端核心静态包
sudo cp -r frontend/ /opt/tdsql-sqlcheck/

# 复制依赖说明文件
sudo cp requirements.txt /opt/tdsql-sqlcheck/

# 复制版本标记文件
sudo cp VERSION /opt/tdsql-sqlcheck/

# 复制新版运维与验证脚本
sudo cp -r deploy/ /opt/tdsql-sqlcheck/
```
> [!IMPORTANT]
> **请勿覆盖** `/opt/tdsql-sqlcheck/.env` 文件以及 `/opt/tdsql-sqlcheck/data/` 目录，以保持原有的数据库连接设置与口令！

### 第 6 步：升级 Python 离线依赖包 (利用内置 wheels)
```bash
cd /opt/tdsql-sqlcheck
sudo ./venv/bin/pip install --no-index --find-links=/tmp/dist/tdsql-sqlcheck-v1.0.4.9-linux-x86_64/wheels/ -r requirements.txt
```
**期望输出**：`Successfully installed` 或 `already satisfied`。

### 第 7 步：启动服务并检查运行状态
```bash
sudo systemctl start tdsql-sqlcheck
sudo systemctl status tdsql-sqlcheck
```
确保服务状态为 `active (running)`，且无报错日志。
