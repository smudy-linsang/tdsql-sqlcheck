gateway_log_analysis 模块变更记录

## interf_batch_analyze.sh v1.1 (2026-04-27)
- feat: 新增 `--instances SPEC [SPEC..]` 参数，支持只分析指定实例（默认行为不变仍为全部）
- feat: SPEC 支持 5 种写法：`#N`/`N`（proxies_N 序号）、`IP:PORT`、`业务名:PORT`、`业务名`（该业务下全部端口）、`IP`（该 IP 下全部端口），多个 SPEC 之间空格分隔取并集
- feat: 未命中任何 SPEC 时自动打印当前配置文件中所有可用实例清单（含 `#序号`、业务名、`IP:端口`）后退出，便于定位
- feat: 脚本 usage 与顶部使用示例同步更新，README 补充 `--instances` 完整用法
- docs: README 版本号同步到 v1.1

## interf_deep_analysis.py v1.6 (2026-04-08)
- feat: sql_explain_schema.csv 新增 6 列诊断信息：EXPLAIN问题标记、索引详情、冗余索引、统计信息更新时间、统计信息是否过期、扫描效率
- feat: 新增 get_index_details() 获取索引详情（索引名、列、类型、基数）
- feat: 新增 get_redundant_indexes() 检测冗余索引（前缀重复索引自动识别）
- feat: 新增 get_stats_update_info() 获取统计信息更新时间及是否过期判断
- feat: 新增 extract_explain_issues() 从 EXPLAIN 结果提取问题标记（全表扫描/文件排序/临时表等）
- feat: 新增 calc_scan_efficiency_from_explain() 计算扫描效率（扫描行数 vs 返回行数）
- refactor: CSV 表头从 15 列扩展到 21 列，与 slow_query_export 模块输出对齐

## interf_deep_analysis.py v1.5 (2026-04-08)
- fix: 修复 EXPLAIN 安全逻辑：UPDATE/DELETE 不再直接 EXPLAIN，改为转写等价 SELECT 后再 EXPLAIN
- feat: 新增 _convert_to_select() 将 UPDATE SET ... WHERE 和 DELETE FROM ... WHERE 转写为 SELECT * FROM ... WHERE
- feat: 新增 _safe_sql_for_explain() 统一安全入口，只允许执行 EXPLAIN SELECT
- fix: INSERT/REPLACE/DDL/SET/SHOW 等非查询语句一律跳过 EXPLAIN
- fix: 拒绝含分号的 SQL（防多语句注入）
- fix: 最终安全校验确保拼接结果严格以 "EXPLAIN SELECT" 开头

## interf_deep_analysis.py v1.4 (2026-04-07)
- feat: 新增 SQL 完整性检测（_is_sql_truncated），自动识别 Proxy 截断的 SQL

- perf: 被截断的 SQL 自动跳过 EXPLAIN 执行和表信息获取，避免无效数据库查询

## interf_deep_analysis.py v1.3 (2026-04-07)
